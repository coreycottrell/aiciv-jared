What “Submission Engine” means (plain-English)

A Submission Engine is a repeatable campaign system that turns a giveaway into three business outcomes:

High-quality UGC at scale (not random “tag 3 friends” junk)

First-party lead capture (email/SMS/opt-ins tied to intent)

Measurable lift (conversion tracking + a clean story for Finance/Legal)

It’s called a “submission engine” because the core mechanic is people submitting something (a story, video, design, receipt, proof-of-play, etc.). Submissions create content, signals, and share incentives—instead of attracting only freebie hunters.

Why anyone would care (and DM you for it)

Your ICP (VP Growth/Marketing) is allergic to:

“Freebie hunters”

“Vanity metrics”

“Legal/compliance headaches”

“Agency deliverables without outcomes”

So the asset you offer must do 3 things:

Show the mechanism (how it actually works step-by-step)

De-risk it (fraud controls + compliance + operational sanity)

Make it defensible (what you’ll measure and how)

When they DM, they’re not asking for “a giveaway idea.”
They’re asking for a system they can take internally and say:
“This is compliant. This filters junk. This captures real demand. This is measurable.”

The DM Asset: Example “Submission Engine Blueprint” (1-page text)

Title: The Experiential Giveaway Submission Engine (CPG + Gaming + Web3)
Purpose: Turn one campaign into (1) UGC pipeline (2) qualified opt-ins (3) measurable lift

1) The Core Loop (the mechanism)

Prompt → Submit → Curate → Vote/Share → Reward → Convert → Retarget → Re-run

Prompt: A question/challenge that people want to answer publicly
Submit: Story / photo / video / design / proof-of-play (not just an entry form)
Curate: Gallery + featured finalists (so it feels like a spotlight, not a lottery)
Vote/Share: Social actions that amplify the submission, not the brand ad
Reward: Winners + “secondary wins” (discounts, access, status, featured placements)
Convert: Opt-in + offer + store locator / purchase path
Retarget: Use submission behavior as intent signals for follow-up
Re-run: Same engine, new prompt, every month/quarter

2) What’s inside the Engine (the build blocks)

A) Entry + Capture (lead quality focus)

2-step entry:
Step 1 = submit (friction = quality)
Step 2 = opt-in (capture happens after intent is proven)

Quality gates (anti-freebie):
“Tell us why / show us / prove it” requirements

Segmentation tags:
industry, persona, product interest, purchase window

B) UGC Gallery (the amplification layer)

Public gallery page with filters (most popular / newest / staff picks)

Finalists page with share cards

“Share-to-unlock” bonus entries (optional)

C) Fraud + Fairness Shield (the trust layer)

Device + velocity checks (stop bot farms)

Duplicate detection + suspicious pattern rules

Eligibility rules + AMOE language

Public winner page (transparent credibility)

D) Conversion Layer (how it drives revenue)

Instant “thank you” offer (coupon, bundle, early access)

Store locator / “where to buy” baked in

Post-campaign email/SMS sequence:

Day 0: confirmation + reward mechanic

Day 2: featured submissions + social proof

Day 5: offer + purchase path

Day 10: winner reveal + “next drop” tease

E) Measurement (Finance-proof)

Pick 2–3 primary KPIs:

Cost per qualified opt-in

UGC volume + usable rate

Conversion rate (and/or retail lift proxy)
Optional: holdout geo / code redemption / click-to-store-locator

3) Three example prompts (so you see it’s not generic)

Example 1 — CPG “Wild Moment” engine

Prompt: “What’s the most chaotic moment a snack/drink should have appeared and saved the day?”
Submission: short story + photo/video
Gallery: “Saved by ____” wall (people vote on funniest/realest)
Reward: grand prize experience + secondary weekly wins
Conversion: everyone gets a “thank you” offer + store locator

Example 2 — Gaming “Avatar → Real Life” engine

Prompt: “Design an outfit/skin/capsule—community votes—winning design goes live.”
Submission: design + short description
Reward: in-game release + creator credit + optional real-world activation
Conversion: newsletter/discord opt-in + drop countdown

Example 3 — Web3 “Proof-of-Participation” engine (no cringe)

Prompt: “Show how you used the product to do X; best proofs win access/status.”
Submission: proof clip/screenshot + explanation
Reward: access tier, whitelist, merch, event seat
Conversion: opt-in + invite to community + follow-up activation

4) Implementation timeline (so it feels real)

Week 1: prompt + rules + landing + form + fraud filters

Week 2: gallery + voting + share assets + email/SMS flows

Week 3–4: run campaign + weekly winners + final reveal

5) Quick “Fit Check” (who this works best for)

Best if you have:

a launch, seasonal window, event, or community moment in the next 90 days

a need for UGC + list growth + measurable outcomes

a brand that wants “earned attention,” not just paid impressions

CTA: If you want me to score-fit this for your next campaign, drop it here: [Fit Check link]

What you’re getting DMs for (in one sentence)

People DM you ENGINE to get a 1-page blueprint that shows exactly how to run a compliant, anti-freebie, measurable experiential giveaway that generates UGC + qualified leads—with examples they can copy.

function install() {

  const ss = SpreadsheetApp.getActiveSpreadsheet();

  // Remove existing triggers for this function to prevent duplicates

  const triggers = ScriptApp.getProjectTriggers();

  triggers.forEach(t => {

    if (t.getHandlerFunction() === "onFormSubmit") ScriptApp.deleteTrigger(t);

    .onFormSubmit()

    .create();

/**

function onFormSubmit(e) {

  const sheet = e.range.getSheet();

  const row = e.range.getRow();

  if (TARGET_SHEET_NAME && sheet.getName() !== TARGET_SHEET_NAME) return;

  const headers = sheet.getRange(1, 1, 1, sheet.getLastColumn()).getValues()[0];

  const rowValues = sheet.getRange(row, 1, 1, sheet.getLastColumn()).getValues()[0];

  const norm = (s) => String(s || "").trim().toLowerCase();

  const findIdx = (name) => {

    const exact = headers.indexOf(name);

    if (exact !== -1) return exact;

    const target = norm(name);

    return headers.findIndex(h => norm(h) === target);

  };

  const idx = (name) => findIdx(name);        // 0-based

  const col = (name) => idx(name) + 1;        // 1-based

  const val = (name) => rowValues[idx(name)];

  const required = [

    "Email Address",

    "Total Score",

    "Tier",

    "Email Sent?",

    "Email Sent At",

    "Email Template Sent"

  ];

  for (const h of required) {

    if (idx(h) === -1) throw new Error(`Missing required header: "${h}" in Row 1.`);

  }

  const email = val("Email Address");

  const scoreRaw = val("Total Score");

  const tierRaw = val("Tier");

  const alreadySent = norm(val("Email Sent?")) === "yes";

  if (!email || alreadySent) return;

  const score = Number(scoreRaw) || 0;

  const tier = String(tierRaw || "").trim();

  const { subject, htmlBody, templateName } = buildEmailTemplate({ score, tier });

  MailApp.sendEmail({

    to: email,

    subject,

    htmlBody
