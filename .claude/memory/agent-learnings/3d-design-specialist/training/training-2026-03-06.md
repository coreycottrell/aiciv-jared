### 2.2 Three.js + Drei — New Techniques From March 2025 Research

**Codrops "Warping 3D Text Inside a Glass Torus" (March 2025)**:

Three underutilized Drei MeshTransmissionMaterial properties we have not yet fully explored:

1. **`distortion` property**: Warps light waves traversing the material. Creates heat-shimmer or water effects. Different from `temporalDistortion` — static warp vs animated warp. Range: 0.0-0.5.

2. **`temporalDistortion` without useFrame**: The distortion animates internally — no animation loop required. `temporalDistortion={0.35}` auto-animates the FBO UV coordinates.

3. **IOR-based dramatic warping**: Animate IOR values (1.07-1.5 range) via GSAP timeline to create "wooshing" refraction shifts without any shader code. Underused technique.

```jsx
// IOR animation with GSAP (npm track only)
gsap.to(materialRef.current, {
  ior: 1.08,
  duration: 0.6,
  ease: "power2.out"
})
// Then back to 1.52
gsap.to(materialRef.current, {
  ior: 1.52,
  duration: 1.2,
  ease: "elastic.out(1, 0.8)"
})
```

**Performance note from research**: Multiple transmission materials force scene re-renders per transmission object. For production: limit to 1-2 transmission objects in foreground; use non-transmission glass approximation for background/secondary objects.

### 2.3 GSAP ScrollTrigger + Three.js — Cinematic Scroll Architecture

**Critical new pattern for purebrain.ai**: Codrops November 2025 cinematic scroll tutorial.

**Core architecture** (scroll-connects-to-camera):

```javascript
const cameraAnimRef = useRef({ x: -20, y: 0, z: 0 })
const targetAnimRef = useRef({ x: 0, y: 15, z: 0 })

// ScrollTrigger scrubs a GSAP timeline
const tl = gsap.timeline({
  scrollTrigger: {
    trigger: containerRef.current,
    start:  "top top",
    end:    "bottom bottom",
    scrub:  true  // scroll velocity = animation velocity
  }
})

// Map scroll segments to camera positions
scenePerspectives.forEach((perspective) => {
  const start = perspective.scrollProgress.start / 100
  const end   = perspective.scrollProgress.end / 100
  tl.to(cameraAnimRef.current, {
    x: perspective.camera.x,
    y: perspective.camera.y,
    z: perspective.camera.z,
    duration: end - start,
    ease: "none"  // scrub handles easing
  }, start)
})
```

**Key insight from research**: `scrub: true` eliminates RAF timer conflicts — scroll velocity directly drives animation progress. No `useFrame` fighting with `gsap.to` timers.

**ScrollSmoother pattern** (canvas stays fixed, content scrolls over it):

```javascript
ScrollSmoother.create({
  wrapper: smoothWrapperRef.current,
  content: smoothContentRef.current,
  smooth: 4,
  effects: false
})
// Canvas = position:fixed background
// Content div = scrolls via ScrollSmoother
```

**Custom cinematic easing curves**:
```javascript
CustomEase.create("cinematicSilk",   "0.45,0.05,0.55,0.95")
CustomEase.create("cinematicSmooth", "0.25,0.1,0.25,1")
```

**Hero section strategy** (0-15% of total scroll):

This creates cinematic context BEFORE narrative content begins.

## PART 4: CRITICAL EVALUATION

### Performance Risks and Mitigations

**Risk 1: Transmission material performance cliff**

Multiple `MeshPhysicalMaterial` with `transmission:1` creates O(n) extra render passes.

- Mitigation: Limit to 1-2 transmission objects per scene
- Mitigation: Use dual-layer approximation (BackSide+FrontSide without FBO) for background objects
- Mitigation: On mobile, drop to single-layer glass (80% quality, 2x performance)

**Risk 2: Integrated GPU laptops**

~30fps on scene with 3 transmission objects + full postprocessing.

- Mitigation: Quality detection at load time: `gl.getExtension('WEBGL_draw_buffers') && devicePixelRatio`
- Mitigation: Adaptive quality: disable DoF on mid-range, disable CA on low-end
- Rule: Never drop below 30fps. If scene is under 30fps at 1x DPR, reduce technique count.

**Risk 3: WordPress CSP headers blocking WebGL**

purebrain.ai on GoDaddy/Cloudflare may block canvas WebGL.

- Known solution: Security plugin CSP configuration (full-stack-developer domain)
- Pattern from memory: `threejs-csp-connect-src-dynamic-import-fix.md` — exact headers documented
- Always test after WordPress plugin update

**Risk 4: WebGL not supported (< 2% of users)**

```javascript
// Graceful fallback
function detectWebGL() {
  const canvas = document.createElement('canvas');
  return !!(canvas.getContext('webgl2') || canvas.getContext('webgl'));
}

if (!detectWebGL()) {
  document.getElementById('threejs-container').innerHTML =
    '<img src="/assets/3d-fallback.jpg" class="static-hero">';
}
```

- **Phones**: Single-layer glass, no DoF, bloom only, max 1,500 particles
- **Tablets**: Dual-layer glass, bloom + CA, max 3,000 particles

## Gotchas Catalogue (Permanent Reference)

### Gotcha 1: `depthWrite: false` on ALL transmission materials in multi-glass scenes

Non-negotiable. Transmission materials fight for depth test. Always set `depthWrite: false`.

### Gotcha 2: `renderer.clearDepth()` between background scene and main scene

Without it: background depth buffer occludes all main scene fragments — blank screen.

### Gotcha 3: ExtrudeGeometry hex requires `geo.center()` and `-Math.PI/6`

Without center: geometry offset from origin. Without -PI/6: pointy-top instead of flat-top.

### Gotcha 4: SMAA must match `devicePixelRatio`, not CSS canvas size

```javascript
const smaaPass = new SMAAPass(
  innerWidth * renderer.getPixelRatio(),
  innerHeight * renderer.getPixelRatio()
);
```

### Gotcha 5: N8AO before SMAA in composer chain

AO must run on raw render, before anti-aliasing. Chain: `Render → N8AO → SMAA → Bloom → CA → Output`

### Gotcha 6: `geo.computeVertexNormals()` required after ExtrudeGeometry

Without it: flat-shaded faces, normals don't work correctly for glass materials.

### Gotcha 7: Bloom luminanceThreshold ≥ 0.82 — never lower

Lower threshold = bloom on non-emissive regions = washed-out scene. Gleb's bloom is surgical.

### Gotcha 8: Background must be `#060606` minimum, never `#000000`

Pure black kills atmospheric light bleed from sphere. The faint dark hue distinguishes "premium dark" from "broken dark."

### Gotcha 9: GSAP `scrub: true` resolves RAF conflicts in R3F

Without scrub, GSAP animations fight with `useFrame`. Scrub makes scroll the clock — no conflict.

### Gotcha 10: Meshy generation is async — never assume instant completion

Poll every 10 seconds. Status: `PENDING → PROCESSING → SUCCEEDED / FAILED`. Budget 3-5 minutes.

### Gotcha 11: WordPress CSP may block dynamic Three.js imports

If white/blank canvas in WP: check `threejs-csp-connect-src-dynamic-import-fix.md` for exact header config.
