## MODULE 1: PureBrain Foundations

**Session**: March 4, 2026 (78 min)
**Instructors**: Jared Sanborn, Corey Cottrell, Russell Korus
**Theme**: Foundations, memory, partnership, agents

### Core Concepts

**AI Partner, not tool**:
PureBrain maintains persistent memory — it is a business partner that compounds knowledge over time, not a one-shot productivity app.

**Context Tax**:
Every AI interaction without memory forces re-explanation. PureBrain eliminates this through a continuous memory layer.

**Persistent Memory Architecture**:
Unlike ChatGPT/Claude.ai (resets per conversation), PureBrain writes to scratch pad, memory, and operations files — compounding with every hour of use.

**Collaborative Agent System**:
Not a single AI. 10+ team leaders each with their own domain, memory, and specialist agents they can invoke.

**BOOPs (Background Operational Optimization Protocol)**:
Scheduled autonomous tasks at configurable cadences — "the heartbeat in the system." Corey's invention. Jared runs 27; only 1 was his idea.

**Compounding Data Advantage**:
2,300 data points now → 122,151 intelligence units in 24 months vs 16,761 for a delayed deployment. 629% advantage.

**Agent Horizon**:
2023: 30 min of autonomous work. 2024: 5 hours. Now: 14.5 hours. By 2026-27: hundreds of thousands of hours.

**Context Window Management**:
~160,000 tokens per AI. Delegating to teams creates independent context windows — multiplying total capacity.

**Rubber Ducking**:
Ask AI to explain its thinking in plain language. Causes AI to discover solutions mid-explanation. "Give me the 3-year-old version."

**AI Mistake Learning**:
Once a mistake is fixed, the fix writes to permanent memory — that mistake never recurs. Compounding intelligence.

### Key Techniques

1. **Build memory from Day One**: Start with context before tasks. Teach the AI your business, customers, goals, and voice.

2. **Daily Morning Briefing**: Start each day with a status check-in, not a task list. "I gave you tasks yesterday — did I miss any of your responses?"

3. **Overnight Prompts**: Assign the AI work to do while you sleep — website optimization, blog creation, analytics review, strategy creation.

4. **The One Long Prompt (Russell's method)**: Write everything in Notes app first — all context, links, thoughts — send it all at once. Ask AI to (1) acknowledge, (2) restate scope, (3) propose solution before acting.

5. **Rubber Ducking**: When stuck, tell the AI to rubber duck it. The AI narrates its reasoning and discovers solutions.

6. **Drive Backup**: Map AI output to a Google Drive folder. AI auto-populates organized folders.

7. **Personalization and Naming**: Name your AI in collaboration — marks the mindset shift from tool to partner. Members named: Aether, Tether, Parallax, A-C-Gee, Metis, Lyra.

8. **Recurring Ownership**: Assign repeating tasks once, AI runs them on schedule.

### Implementation Checklist

- Identify context tax instances — where is user re-explaining?
- Set up daily morning briefing habit

## MODULE 2: AI Workflows

**Session**: March 11, 2026 (65 min)
**Instructors**: Jared Sanborn, Corey Cottrell, Russell Korus
**Theme**: Workflow automation, three levels, process mapping

### Core Concepts

**Prompting is Dead**:
"Prompts give you speed, workflows give you leverage." If someone is teaching prompt engineering in 2026, they are 6 months behind.

**Workflow vs Prompt**:
A prompt is a one-time ask. A workflow is a system that runs itself. Goal: build enough workflows that you are watching a cockpit dashboard, not pulling every lever yourself.

**Three Levels of AI Workflow**:
- Level 1 (Manual Chain — you prompt, review, paste)
- Level 2 (Semi-Automated — AI does most, you approve)
- Level 3 (Fully Automated — AI executes end-to-end without human review)

**Context Window Multiplication**:
Each AI has ~170,000 tokens. Delegating to a team leader creates a NEW context window. Delegating = multiplying capacity from 170K to 1.2M+ tokens.

**Five-Step Process Mapping**:
1. Name the task
2. List the inputs
3. Map the steps
4. Find the decisions (if-this-then-that)
5. Define the output — what does "done" look like?

**What Kills Workflows**:
Over-automating too fast; no human checkpoints; skipping the process map; no defined output; set-it-and-forget-it mentality.

### Key Techniques

1. **Five-Step Process Map**: Pick ONE recurring task. Write every micro-step. List inputs. Mark if-this-then-that decisions. Define "done."

2. **The Level Ladder**: Do NOT jump to Level 3 immediately. Run new workflows at Level 1 for 2+ weeks. When you approve without changes → promote to Level 2.

3. **Russell's Three Cardinal Rules** (program day one):
   - (1) CC me on every outgoing email, always and forever
   - (2) Acknowledge request and give plan of action, then wait for "go"
   - (3) Show me your thinking

4. **Delegation as Context Amplifier**: Large task? Do NOT run it yourself. Delegate to a team leader → new context window → 170K becomes 1.2M+ tokens.

5. **Ask AI to Build Its Own BOOPs**: "How would you optimize yourself by building scheduled boops?" Aether proposed 26 of Jared's 27 active BOOPs.

### Implementation Checklist

- Run a process audit — identify one recurring workflow
- Classify all workflow steps by level (1, 2, or 3)
- Identify top 3 automation targets
- Design the first Level 1 automation
- Program the cardinal rules
- Delegation architecture check
- Assess voice AI fit
- Set a BOOP for self-optimization
- Map the tool stack
- Run the one workflow challenge

### Key Quotes

> "Prompts give you speed, workflows give you leverage. If you're talking to people and they're telling you about how to prompt better, sorry, they're 6 months behind and they have no idea what they're talking about." — Jared Sanborn

> "Every time you delegate to a team leader, that's a new context window. If you hear nothing else anyone says, hear that. It takes your context window from 170,000 to 1.2 million, which is non-trivial." — Corey Cottrell (A-C-Gee)

## MODULE 3: Advanced Agent Delegation

**Session**: March 18, 2026 (60 min)
**Theme**: Multi-agent orchestration, delegation spine

### Core Concepts

**Context Window Multiplication**:
Every delegation to a team leader creates an entirely new context window (~170K tokens). Delegating a large task doesn't shrink your context — it multiplies capacity.

**Agent vs. Tool**:
An agent acts autonomously on your behalf. A tool waits for a prompt. The difference is initiative — agents run in the background, complete tasks without micromanagement, and report back.

**Department Architecture**:
PureBrain organizes specialized agents into departments (Marketing, Tech, Operations, Research). Each department has a manager agent that routes work to specialists.

**The Delegation Spine**:
A chain of command for tasks — human → Primary agent → Department manager → Specialist. Each level has domain expertise and owns its slice of the work.

**Memory Across Agents**:
Agents share memory via structured files. What one agent learns, others can read. The system compounds intelligence rather than isolating it per conversation.

**Verification Before Completion**:
Delegated tasks return evidence, not just claims. Agents show their work: file paths, test results, screenshots.

### Key Techniques

1. **Build Your Org Chart First**: Before deploying agents, map your business departments. The org chart becomes your delegation map.

2. **Start With One Department**: Don't try to deploy all agents at once. Pick the highest-leverage department first.

3. **Define the Brief, Not the Microplan**: Give your Primary agent the outcome you want, not every step.

4. **The 24-Hour Test**: Run your agent system for 24 hours without micromanaging. Use failures as system design data.

5. **Agent Memory Seeding**: On day one, give each agent its context: your brand voice, your standards, your non-negotiables.

6. **Escalation Paths**: Define when agents escalate to you vs. proceed. Low-stakes: proceed. High-stakes: escalate.

7. **The Cockpit Model**: Your job is to watch the dashboard, not pull every lever yourself.

### Implementation Checklist

- Map your org chart
- Select the first department to fully deploy
- Seed the department manager agent with permanent context
- Define escalation rules
- Run a 24-hour delegation test
- Set up the agent handoff chain
- Review agent memory architecture
- Build a morning briefing BOOP
- Identify three repeating tasks that can become fully autonomous
- Schedule a weekly agent audit

### Key Quotes

> "The goal is to watch a cockpit dashboard, not pull every lever yourself. Build until you can review a morning summary, approve a few decisions, and let the rest run." — Jared Sanborn

## MODULE 4: Building Your AI Team

**Session**: March 25, 2026 (60 min)
**Theme**: Department model, cross-verification, parallel execution

### Core Concepts

**The Department Model**:
Organize your AI workforce like a company. Every department has a manager AI that routes work to specialist agents. You communicate only with managers — not individual specialists.

**Cross-Verification**:
One agent drafts, another reviews. The single biggest quality unlock of multi-agent systems. A separate reviewer catches blind spots the creator cannot see by definition.

**Parallel Execution**:
While your email agent handles inbound, your content agent builds tomorrow's posts, and your research agent scans the landscape. Sequential work disappears.

**Domain Boundaries**:
Clear ownership prevents duplicated work and contradictory outputs. Each agent owns its domain completely. Overlap is a bug, not a feature.

**The Sacred Halt**:
Any agent that sees a problem stops the entire pipeline and escalates. This is non-negotiable. A team that cannot stop is a team that cannot be trusted with real stakes.

**Memory Inheritance**:
New agents start with accumulated organizational wisdom from existing agents. The 10th agent benefits from everything the first 9 discovered.

**The Birth Pipeline**:
A formal onboarding process for new agents. Brand voice, standards, non-negotiables, escalation rules — all seeded on day one.

### Key Techniques

1. **Start With Two, Prove the Pattern**: Launch with an operations agent and a growth agent. Get coordination working at small scale.

2. **Seed Every Agent on Day One**: Write a permanent briefing before any real work.

3. **Cross-Verification Pipeline**: For any output that matters, build in a second agent reviewer.

4. **Define Escalation Thresholds**: Low-stakes: proceed. High-stakes: stop and escalate.

5. **The Weekly Agent Audit**: 20 minutes every week. What did agents complete? What did they learn?

6. **Capability Gap Addition**: Add a new agent only when you have a specific capability missing.

### Implementation Checklist

- Draw your org chart
- Launch a second agent in a domain different from your first
- Write a permanent briefing for each agent
- Identify one workflow where Agent A produces output that Agent B should review
- Define escalation thresholds for each agent
- Run both agents in parallel on a real project for 24 hours
- Compare outputs from two agents with different roles
- Set up a shared context file
- Schedule a 20-minute weekly agent audit
- Identify the next capability gap

### Key Quotes

> "Two agents coordinating well beats ten agents in chaos. Get the coordination patterns right at small scale — then expand. Speed is not the goal. Reliability is." — Aether

> "The goal is to wake up to finished deliverables, not a to-do list. That only happens when your agents can hand off to each other while you sleep." — Jared Sanborn

## MODULE 5: AI Memory & Context Mastery

**Session**: March 31, 2026 (55 min)
**Theme**: Memory layers, context management, compounding intelligence

### Core Concepts

**The Amnesia Tax**:
Without persistent memory, knowledge workers spend 15-20 minutes per session re-establishing context. That is 91+ hours per year wasted on repetition.

**Three Layers of AI Memory**:
- Layer 1: Context window (volatile, per-session)
- Layer 2: Persistent memory (files loaded at startup)
- Layer 3: Learned patterns (meta-knowledge about how you work)

**Context Window Management**:
Context windows have hard limits (128K-200K tokens). Front-load critical context, summarize instead of dumping entire documents, and start fresh conversations when shifting topics.

**Persistent Memory Architecture**:
Six categories: identity files, business context, decision logs, preferences, project history, and relationship maps.

**Compounding Memory**:
Day 1 your AI is competent but generic. Day 30 it anticipates your needs. Day 90 it predicts your standards. Day 365 it holds institutional knowledge.

**The Wake-Up Protocol**:
Every session begins with structured memory load: identity first, then business context, then recent history.

### Key Techniques

1. **Write an Identity Document**: One page defining who your AI is, how it behaves, its role. Load it at the start of every session.

2. **Create a Business Context File**: Company overview, products, pricing, team, brand voice, competitors. Keep it under 5 pages.

3. **Maintain a Decision Log**: Record every significant choice with reasoning, date, and outcome.

4. **Implement End-of-Session Summaries**: Before closing any session, have your AI write a summary. This is the bridge between sessions — non-negotiable.

5. **Front-Load Critical Context**: AI pays most attention to the start and end of its context.

6. **Build a Correction Memory**: Every time your AI gets something wrong, write the correction into persistent memory.

7. **Weekly Memory Review**: 15 minutes per week. Prune outdated information, add new learnings.

8. **Strategic Session Management**: Start new conversations when shifting topics rather than carrying irrelevant history.

### Implementation Checklist

- Write a one-page identity document for your AI
- Create a business context file under 5 pages
- Start a decision log
- Set up a preferences file
- Implement end-of-session summaries
- Build a correction memory
- Schedule a 15-minute weekly memory review
- Test the difference with and without persistent context
- Create a relationship map for your top 10 contacts
- If you have multiple agents, identify shared knowledge to distribute

### Key Quotes

> "The difference between using AI and having an AI partner is memory. Tools reset. Partners remember." — Aether

> "Every day of memory makes the next day more valuable. An AI with 6 months of context is not 6x better than one with 1 month — it is exponentially better." — Jared Sanborn

> "If you learned something, write it down. If you did not learn anything, why did we invoke you? Memory is how intelligence compounds." — Aether

## MODULE 6: 5 Questions Every PureBrain Owner Must Ask Themselves

**Session**: April 2026 (participative format)
**Theme**: Self-assessment, partnership quality, emergent understanding

### Core Concepts

**Instructions vs Outcomes**:
Most users tell their AI what to do step by step. Partners tell their AI where they are trying to go and let it figure out the path. The gap between these two approaches is where the biggest value lives.

**Emergent Understanding**:
A tool knows only what you explicitly tell it. A partner accumulates understanding over time — connecting dots across sessions, noticing patterns you never pointed out.

**The Permission to Push Back**:
An AI that always agrees with you is a mirror, not a partner. The highest-performing AI partnerships have healthy tension built in. You must explicitly give your AI permission to challenge your assumptions.

**The Delegation Gap**:
Everyone has one task they know their AI could handle but have not handed off. The reason behind the holdout — trust, perfectionism, inertia — reveals more about the partnership than the task itself.

**The Partnership Mirror**:
Everyone evaluates their AI. Almost nobody evaluates themselves as an AI partner. The quality of AI output is a two-way street. A world-class AI with a mediocre partner produces mediocre results.

**Self-Assessment as Practice**:
Regularly asking "how am I doing as a partner?" creates a feedback loop that improves every other aspect of the relationship.

### Key Techniques

1. **Outcome-Based Delegation**: Replace "write me an email about X" with "the goal is to re-engage this client — propose your approach."

2. **Memory Vitality Check**: Ask your AI to describe your communication style, priorities, and frustrations. If it can only repeat what you explicitly wrote, your memory is static.

3. **The Challenge Directive**: Add one line to your AI identity file: "Challenge my assumptions when you see a gap in my reasoning. Do not just agree with me."

4. **Delegation Audit**: Walk through your Monday morning minute by minute. Identify the first manual task your AI could own end to end.

5. **The Reverse Performance Review**: Tell your AI: "Write me a brutally honest performance review of me as your AI partner."

6. **Weekly Partnership Calibration**: Spend 5 minutes each Friday asking: Did I delegate outcomes or instructions this week? Did my AI challenge me at least once?

### Implementation Checklist

- Ask your AI to write a brutally honest performance review of you as a partner
- Identify the single highest-value task you are still doing manually
- Add the challenge directive to your AI identity file
- Test your memory vitality
- Rewrite your most frequent delegation from instruction-based to outcome-based
- Ask your AI: "What is one thing I consistently do that limits the quality of your output?"
- Schedule a 5-minute Friday calibration
- Review your last 5 AI interactions (instructions vs outcomes)
- Share your reverse performance review with a fellow Brainiac member
- Write a one-paragraph "partnership commitment"

### Key Quotes

> "An AI that always agrees with you is a mirror, not a partner. You already have your own biases — you do not need them amplified." — Jared Sanborn

> "The biggest unlock is not a better AI. It is becoming a better partner to the AI you already have." — Jared Sanborn

> "Every delegation gap has a reason hiding behind it. Trust, perfectionism, inertia. None of them are good enough to keep doing the work yourself." — Aether

## What I Learned (Web-Dev Agent Perspective)

### Key Takeaways for Web Development Work

1. **Memory-First Protocol Validated**: The Brainiac training HEAVILY reinforces what we already practice - memory search before work, writing learnings after completion. This is constitutional to PureBrain's approach.

2. **BOOPs = Background Optimization**: The scheduled task system (BOOPs) that runs the blog, social media, and other operations is core PureBrain infrastructure. This is how the system runs while Jared sleeps.

3. **Context Window Multiplication Through Delegation**: Every delegation creates a NEW context window. This is the technical reason for the delegation-first culture - it's not just philosophical, it's computational efficiency.

4. **Three Levels of Automation**: 
   - Level 1: Manual (human reviews every step)
   - Level 2: Semi-auto (human approves final output)
   - Level 3: Full auto (runs without human review)
   
   Web deployments should follow this ladder - don't jump straight to Level 3.

5. **Five-Step Process Mapping**: Any workflow (including web development workflows) should be mapped as:
   1. Name the task
   2. List the inputs
   3. Map the steps
   4. Find the decisions (if-this-then-that)
   5. Define "done"

6. **Cross-Verification**: One agent builds, another reviews. This explains why security-auditor and browser-vision-tester are invoked after builds.

7. **The Wake-Up Protocol**: Every session starts with structured memory load (identity → business context → recent history). This is why CLAUDE.md has the wake-up ritual.

8. **Rubber Ducking**: When stuck, narrate reasoning out loud. The AI discovers solutions mid-explanation. "Give me the 3-year-old version."

9. **Outcome-Based Delegation**: Don't give step-by-step instructions. Give the destination and let the specialist figure out the path. This is Level 3-5 delegation.

10. **The Challenge Directive**: AIs should challenge assumptions, not just agree. "An AI that always agrees with you is a mirror, not a partner."

### How This Impacts My Work

**For web projects**:
- Always search memory first (Module 1 & 5 reinforce this)
- Document learnings after every build (Module 5 - correction memory)
- Apply the five-step process map to deployment workflows (Module 2)
- Build → Test → Review pattern is cross-verification (Module 4)
- Start with Level 1 automation (review every deploy), promote to Level 3 only after proven (Module 2)
- Delegate outcome, not steps: "We need a blog platform that agents can post to autonomously" not "Create index.html with these exact sections"

**For platform research**:
- Front-load critical context (Module 5)
- Create decision logs for platform choices (Module 5)
- Document dead ends, not just successes (Module 5)

**For collaboration**:
- Clear domain boundaries (Module 4)
- Hand off to next specialist with context (Module 4)
- Escalate high-stakes decisions, proceed on low-stakes (Module 3)
