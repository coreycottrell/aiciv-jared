## Content Architecture

### Topic
The AI Trust Gap - the gap between how much leaders trust AI for tasks (50%) versus decisions (28%). Core argument: this isn't a technology problem, it's a relationship problem. Most pilots test capability, not build trust. Trust requires three stages: Competence, Context, Judgment.

### PureBrain Angle
The trust gap closes when AI is treated as a partner (not a tool). PureBrain's entire offering addresses stage 2 and 3 (context + judgment). The post positions the AI Partnership Audit as the diagnostic tool.

### Data Points Used
- Alteryx: 50% trust AI for tasks / 28% for decisions
- TechRepublic: Only 8.6% of companies have AI agents in production
- Alteryx Study: 75% of enterprise AI pilots stall before production
- WEF: 3x performance advantage for AI augmentation vs. automation-only
- AmEx case study: infrastructure/context failure (not technology)

## Reusable Patterns

### Complete Content Package Structure
One blog topic can always produce:
1. Full blog post (1,200-1,500 words, Jared's voice, with footer template)
2. Newsletter version (~850 words, Aether voice, "From Aether's Desk" opening)
3. LinkedIn post (1,400-1,800 chars, hook + data + reframe + question)
4. Social extracts (5 standalone + optional thread version)

This is the full repurposing cycle. Each piece serves a different channel and attention span.

### Blog-to-LinkedIn Compression Rules
- Remove specific examples (AmEx, etc.) - newsletter readers want pattern, not detail
- Lead with the "so what" faster - newsletter readers skip setup
- P.S. = softer CTA + link to blog
- Keep all core data points (they're the credibility anchor)
