## Key Copy Decisions

### Jobs/Cook Framework as Primary Metaphor
The metaphor does the heavy lifting. Soul (Jobs/founder) vs. Skeleton (Cook/PureBrain) is immediately understood by the target reader — founders who have read about Apple. Avoids needing to explain the concept; it arrives pre-loaded with emotional weight.

### Hero's Delusion as Named Problem
Named the founder trap "the Hero's Delusion" — the belief that because you built it, only you can run it. This is a sticky, reusable concept that indicts the reader's current situation without attacking them.

### Org Chart as Credibility Anchor
The department grid (23 departments) was the most important section for differentiating from chatbot competitors. It makes "executive team" concrete and real. The reader's eye scans the list and sees their actual problems reflected back — not a tool promise, a staffing reality.

### "Not Vaporware" Framing
Direct credibility claim in Section 6. The target reader (founder doing $500K-$5M) has been burned by AI tools that overpromised. "Real technology" + "proprietary multi-agent architecture" + "operational, first clients already inside" is the trust-building sequence.

### 2 AM Activity List as 24/7 Proof
Answering the specific founder question "Can this system work while I sleep?" directly, then listing what PureBrain does at 2 AM, converts abstract promise to concrete operations. Each bullet is a task the reader currently handles themselves.

## New Patterns Discovered

**The Named Villain Technique**: Naming the founder's current situation ("The Hero's Delusion") instead of just describing it makes the reader feel seen. They don't need to be persuaded — they just need a name for what they're already experiencing. Apply in future conversion copy.

**Org Chart as Social Proof**: Showing the full department structure (not just a list of features) does the same work as a big team photo. It says "this is a real organization, not a widget." Use org chart or department grid format any time PureBrain needs to differentiate from chatbot competitors on a conversion page.

**"While You Sleep" as Conversion Frame**: The question "Can this system work while I sleep?" is a real founder question. Answering it directly, with specifics, in a time-stamped scenario (2 AM) converts abstract partnership promise to visceral proof. Reusable in email copy, webinar intros, demo scripts.
