## Module 01: Foundations of AI Partnership

**Live date**: March 4, 2026
**Duration**: 78 minutes
**Format**: 11-slide interactive presentation deck (keyboard navigable)

### Core Concepts

1. **Persistent Memory Architecture**
   - PureBrain retains user context across sessions (unlike standard ChatGPT/Claude)
   - Eliminates the "context tax" - no repeating yourself every session
   - Two memory tiers: short-term (session) and long-term (permanent)
   - This is the foundational differentiator from commodity AI tools

2. **BOOPs (Background Operational Optimization Protocols)**
   - Autonomous background operations that handle routine tasks without micromanagement
   - AI runs scheduled/triggered tasks without user prompting each time
   - Reframes AI from "tool you use" to "partner that acts"

3. **Context Windows and Compacting**
   - Context window = working memory of the current session
   - Compacting = summarizing and preserving key context before session ends
   - Understanding this prevents "AI amnesia" frustration

4. **How to Talk to Your AI**
   - Specificity over vagueness - precise inputs yield precise outputs
   - Share context proactively - treat AI as a colleague who needs briefing
   - Avoid treating it as a search engine (query-response model is wrong mental model)
   - Right mental model: briefing a capable team member

5. **Rubber Ducking**
   - Explain your problem aloud (to AI) to clarify your own thinking
   - The act of articulation reveals solutions
   - AI as thinking partner, not just answer machine

6. **Status Monitoring and Delegation**
   - Regular check-ins with AI agents (not set-and-forget)
   - Task ownership delegation - assign clearly, confirm receipt
   - AI should report back proactively

7. **The Compound Effect**
   - Value curve: Week 1 (basic utility) → Month 1 (workflow integration) → Month 6+ (exponential leverage)
   - Investment in training AI on your context pays compound returns
   - Early adopters who train their AI now will be unmatchable in 18 months

### Key Techniques

- **Morning briefings**: Start each session with a context update to AI
- **Decision support**: Use AI to pressure-test decisions before acting
- **Voice consistency**: Train AI on your communication style for drafts/comms
- **Operations automation**: Identify 3 repeatable tasks to automate first

### Key Insight

The foundational mental shift is from "AI as a tool I pick up and put down" to "AI as a partner who knows me, remembers our work, and acts on my behalf." The memory architecture makes this possible - everything else builds on top of it.

## Module 02: Building Your First AI Workflow

**Live date**: March 11, 2026
**Duration**: 65 minutes
**Format**: 12-slide interactive presentation deck

### Core Framework: 5-Step Process Mapping

Convert any repeatable task into an automated workflow:

1. **Name the task clearly** - Give it a precise, unambiguous label
2. **Identify required inputs** - What data/context does this task need to run?
3. **Document sequential steps** - Write out every step in order
4. **Mark decision points** - Where does the process branch? What triggers each branch?
5. **Define completion criteria** - What does "done" look like? How do you verify?

### The 3-Level Automation Model

| Level | Name | How It Works | Human Role |
|-------|------|-------------|------------|
| 1 | Manual Chain | User prompts AI at each step, reviews output | Active at every step |
| 2 | Semi-Automated | AI generates multiple outputs; human approves batches | Batch review only |
| 3 | Fully Automated | Triggers execute end-to-end; humans notified on completion | Exception handling only |

**Strategic principle**: Start at Level 1, graduate to Level 3 only after process is proven reliable.

### Candidate Task Signals (What to Automate First)

A task qualifies for workflow automation if it has:
- Repeatable pattern (you do it the same way each time)
- Predictable inputs (you can define what it needs)
- Rule-based decision logic (if X, then Y - not judgment calls)
- Clear completion state (you know when it's done)

**Best first candidates**: Client onboarding, content creation pipeline, lead qualification, weekly reporting, email triage

### Key Insight

"Prompts give speed; workflows give leverage."

A single prompt speeds up one task. A workflow multiplies your capacity across all instances of that task forever. The shift from prompt-user to workflow-builder is the shift from tactical to strategic AI use.

### Implementation Pitfalls to Avoid

1. **Over-automating too fast** - Automate proven manual processes, not unproven ones
2. **Missing human checkpoints** - Even Level 3 automation needs exception handling
3. **Undefined outputs** - If you can't describe "done," you can't automate it
4. **Skipping process documentation** - You must map the process manually before automating it
5. **Starting at Level 3** - Always validate at Level 1 first

### Key Techniques

- Map your top 3 repeatable tasks this week using the 5-step framework
- Assign each a Level (1, 2, or 3) based on current reliability
- Build Level 1 first, run it 5 times, then graduate to Level 2
- Document inputs and outputs before touching AI automation tools
