## Core Concepts

1. **AI as Partner, Not Tool**
   - PureBrain maintains persistent memory — a compounding business partner, not a single-session utility
   - The shift is paradigmatic: stops being software, starts being a collaborator that learns you daily

2. **Context Tax Problem**
   - Traditional AI resets every conversation — users pay a "context tax" re-explaining their business, goals, and voice each session
   - PureBrain eliminates this through continuous memory layers (scratch pad, memory file, operations file)

3. **Persistent Memory Architecture**
   - Multi-file system: scratch pad + memory + operations file
   - Each interaction compounds intelligence — no session is wasted
   - This is the foundational technical differentiator vs. ChatGPT/Gemini/vanilla Claude

4. **Collaborative Agent System**
   - 10+ specialized domain agents with independent memory
   - Sub-agents invokable within the system
   - Each agent gets experience through invocation (identity formation through use)

5. **BOOPs (Background Operational Optimization Protocol)**
   - Scheduled autonomous tasks creating system intelligence
   - Corey Cottrell's framework — ask AI to propose its own BOOP schedule
   - Aether proposed 26 of Jared's 27 active BOOPs autonomously

6. **Compounding Data Advantage**
   - 629% intelligence multiplier over 24 months vs. delayed deployment
   - Every day of delay = compounding loss of institutional intelligence

7. **Agent Horizon Expansion**
   - 2023: 30 minutes of autonomous work
   - 2024: 5 hours of autonomous work
   - Present: 14.5 hours of autonomous work
   - Trajectory is accelerating — not linear

8. **Context Window Multiplication**
   - ~160K tokens per AI instance
   - Delegation to teams creates independent context windows
   - Multiplies total capacity significantly

9. **Rubber Ducking**
   - AI explains its reasoning aloud, discovering solutions mid-articulation
   - Request "explain this like I'm 3" when stuck — forces clarity

10. **Mistake-to-Memory Loop**
    - Errors, once fixed, write permanently to memory
    - Prevents recurrence across sessions
    - Institutional knowledge of what NOT to do compounds alongside what TO do

## Key Implementation Techniques

1. **Memory-First Approach** — Begin every AI relationship by loading full business context (goals, customers, voice, projects) before any task execution
2. **Daily Morning Briefing Ritual** — Status check-in, not task list; builds compounding context over time
3. **Overnight Prompt Delegation** — Assign work during off-hours; AI runs autonomously, delivers review-ready output by morning
4. **One Long Prompt Method** — Consolidate all context/links in notes app, send atomically; AI acknowledges, restates, then proposes before executing
5. **Rubber Duck Debugging** — Request "3-year-old version" of explanations when stuck; forces solution-finding
6. **Google Drive Backup Architecture** — AI auto-populates organized folders; human-readable secondary layer for all deliverables
7. **Personalization and Naming** — Name the AI collaboratively (Aether, Tether, Parallax, Metis, Lyra); identity accelerates partnership depth
8. **Recurring Ownership Assignment** — Assign repeating tasks once; AI executes on schedule without re-prompting
