## Key Patterns

### purebrain.ai Post Structure

All posts use `content.raw` (NOT Elementor). The structure at the end of each post is:
1. Blog content body
2. `<div class="blog-social-share">` - social share icons
3. `<div class="blog-cta-block">` - CTA with "Start Your AI Partnership" button
4. `<!-- Internal Links -->` + related reading section

**Insertion point**: Before the `blog-cta-block` div.
**Schema append**: At the very end of the content (after internal links).

### jareddsanborn.com Post Structure

Two different structures:
1. **Newer posts** (ID 1069): Same `blog-cta-block` pattern as purebrain.ai
2. **Older posts** (IDs 1056, 1065, 1039, 1060): Simple inline CTAs without div wrappers

For older posts, find insertion point by looking for "Ready to awaken" or "Begin the process at PureBrain" text and inserting before that `<p>` or `<hr>`.

### Slug Differences Between Sites

The jareddsanborn.com slugs don't always match purebrain.ai slugs:
- purebrain ID 480 → jareddsanborn ID 1069 (slug: `ai-pilot-purgatory` not the full slug)
- purebrain ID 98 → jareddsanborn ID 1039 (slug: `what-i-named-my-ai-and-what-happened-next`)
- purebrain ID 373 → jareddsanborn ID 1060 (slug: `most-ai-agents-break-the-moment-you-ask-where-the-data-goes` without `-2`)

**Always list all posts and match by title, not just slug** when working across both sites.

## post 381 Special Case

Post 381 (CEO vs Employee) on purebrain.ai had an existing placeholder FAQ schema from `blog_seo_updater.py`. That placeholder contained only 1 dummy FAQ item. It was removed before adding the full 6-FAQ section.

**How to detect and remove old schema**:
```python
old_schema_start = content.find('<!-- FAQ Schema: auto-generated by blog_seo_updater.py -->')
if old_schema_start > -1:
    old_schema_end = content.find('</script>', old_schema_start) + len('</script>')
    content = content[:old_schema_start] + content[old_schema_end:]
```

## Important: Clean FAQ Content

The FAQ drafts from content-specialist included metadata lines like:
- `*Keywords captured: ...*`
- `*People Also Ask potential: HIGH*`

These metadata lines are internal notes only. Do NOT include them in published content. Strip them out before building HTML.

## API Approach

These are standard WordPress posts (NOT Elementor). Use `content.raw` via REST API:
```python
# Fetch
r = requests.get(f'{site}/wp-json/wp/v2/posts/{id}?context=edit', headers=headers)
content = r.json()['content']['raw']

# Update
r2 = requests.post(f'{site}/wp-json/wp/v2/posts/{id}', headers=headers, json={"content": new_content})
```

No Elementor cache clearing needed - these are standard WP posts.
