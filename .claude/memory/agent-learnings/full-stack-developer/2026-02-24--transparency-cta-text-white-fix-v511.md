## Problem

Jared reported text at the bottom of blog post 879 (your-next-direct-report-wont-be-human) was not white and hard to read. Specifically the `.aether-transparency__cta-text` paragraph ("This is what AI partnership looks like...").

## Root Cause

The `.aether-transparency__cta-text` CSS rule had:
```css
color: rgba(224, 230, 240, 0.65);
```

That 65% opacity makes the already-light color dim — especially when theme or Cloudflare compression changes background context.

## Fix Applied

In `tools/security/purebrain-security/purebrain-security-plugin.php`:
- Changed `.aether-transparency__cta-text` color from `rgba(224, 230, 240, 0.65)` to `#ffffff !important`
- Changed `.aether-transparency__cta-text strong` color from `#e0e6f0` to `#ffffff !important`
- Bumped plugin version from 5.1.0 → 5.1.1
- Added changelog entry

## Key Lesson

**The transparency section is injected via the plugin, NOT stored in post content.** When Jared reports something wrong in the transparency section:
1. Check the plugin PHP file for the CSS rule
2. The transparency section HTML is generated by `pb_maybe_inject_transparency_section()` in the plugin
3. The CSS lives in the `$css` heredoc string around line 3800-3940 of the plugin
