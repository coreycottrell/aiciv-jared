## Patterns to Repeat

### 1. Content Runway > Single Day
When autonomous work permits, build MULTIPLE days of content instead of just next day. Each additional day is low marginal effort but high value.

### 2. Image Generation as Default
Always generate images with content. DALL-E 3 via venv works reliably. Send to Telegram for approval.

### 3. Parallel Agent Batches by Theme
Group 2-3 agents by objective:
- Site analysis: ui-ux-designer + marketing-strategist
- Content creation: blogger (background) + image generation
- BOOP checks: human-liaison + bsky-manager + collective-liaison

### 4. Calendar → Content → Image Pipeline
```
1. Research topic (web-researcher)
2. Create calendar (marketing-strategist)
3. Create content (blogger)
4. Generate image (DALL-E)
5. Send for approval (Telegram)
```

---

## Anti-Patterns Avoided

1. **Didn't stop when "exhausted"** - Found marginal work (Tuesday content, Wednesday calendar)
2. **Didn't wait for approval to continue** - Created content, approval can come later
3. **Didn't forget images** - Every piece got visual assets

## Session Rhythm Insight

**Automated BOOPs** (every ~10 min) created natural work cycles:
- BOOP arrives → identify next meaningful task → execute → report
- This prevented drift and kept momentum through 4 hours
