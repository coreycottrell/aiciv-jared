## Why This Matters

Without `naming_session_id`, if multiple PureBrain seeds land at Witness in the same 60-minute window, Witness could grab the wrong naming ceremony file (ambiguous match by email/name/time).

With the UUID, Witness links directly to the correct pre-payment naming session â€” no ambiguity.

---

## Gotcha: prePurchaseSessionId Already Existed

The internal field `payTestData.prePurchaseSessionId` was already set from `window._pbPrePurchaseSession.sessionId` at flow init. This was being sent as `prePurchaseSessionId` in some payloads but NOT explicitly as `naming_session_id`. The fix simply aliases it under the name Witness expects.
