# TRIO Clean Formatting

Constitutional formatting rule for all TRIO messages (Aether, Chy, Morphe) and any list shared with Jared or humans.

## Rule (Jared directive, 2026-05-20)

When listing people, AIs, items, or any structured data in TRIO, CC, portal, or human-facing messages:

**ALWAYS use clean bullet lists. NEVER use paragraph dumps.**

## Format

### Lists of people/AIs
```
- Anchor — John Smith
- Teddy — Robert Orlowski
- Lyra — Nathan Olson
```

### Status lists
```
- Anchor: connected
- Teddy: pending
- Lyra: key issued, not yet online
```

### Mixed info
```
- Anchor (John Smith) — connected, key verified
- Teddy (Robert Orlowski) — pending setup
```

## Anti-pattern (BANNED)

Never do this:
```
Anchor with John Smith is connected and Teddy with Robert Orlowski is pending and Lyra with Nathan Olson has a key issued but isn't online yet and Pragma with Zafeer Hassan...
```

## When this applies

- TRIO messages containing any list of 3+ items
- CC #general posts with rosters or status updates
- Portal messages to Jared with structured data
- Emails listing multiple AIs or humans
- Any human-facing output with enumerable items

## Why

Jared reads on mobile. Paragraph walls of names are unreadable. Bullet lists scan instantly. This is a permanent formatting rule, not a suggestion.
