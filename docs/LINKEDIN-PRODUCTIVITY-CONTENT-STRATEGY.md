## Part 2: Content Calendar Framework

### Posting Cadence

**Target**: 4-5 posts per week

| Day | Post Type | Notes |
|-----|-----------|-------|
| Monday | Daily Summary | Start week with previous Friday's or Monday's data |
| Tuesday | Single Task Highlight | Deep dive on best task from Monday |
| Wednesday | Behind the Scenes OR ROI Focus | Alternate weekly |
| Thursday | Daily Summary OR Single Task | Based on data quality |
| Friday | Weekly Compilation | Wrap up the week |

**Why this cadence**:
- Consistent enough to build expectation
- Varied enough to not feel repetitive
- Matches natural work rhythm (people check LinkedIn early week + Friday)

### Best Posting Times

**Primary windows** (US-centric for Jared's audience):
- **7:00-8:00 AM EST** - Morning commute/coffee check
- **11:30 AM-12:30 PM EST** - Lunch break browsing
- **5:00-6:00 PM EST** - End of day wind-down

**Avoid**:
- Weekends (25-50% lower engagement)
- After 7 PM (diminishing returns)
- During major holidays/events

### Post Type Mix (Monthly)

| Type | Frequency | Percentage |
|------|-----------|------------|
| Single Task Highlight | 8-10 posts | 40% |
| Daily Summary | 4-6 posts | 25% |
| Weekly Compilation | 4 posts | 20% |
| ROI Focus | 2-3 posts | 10% |
| Behind the Scenes | 2-3 posts | 10% |

### Hashtag Strategy

**Core hashtags** (use on every post):
- #AIProductivity (niche, owned)
- #WorkSmarter (broad, engagement)

**Rotating hashtags** (2-3 per post, based on content):
- #ProductivityTips
- #AITransformation
- #TechLeadership
- #FutureOfWork
- #StartupLife
- #SaaS
- #DevProductivity
- #ROI
- #RealResults

**Industry-specific** (when relevant):
- #MarTech
- #SalesTech
- #DataDriven

**Rule**: 4-6 hashtags per post. More looks spammy, fewer loses discoverability.

### Engagement Tactics

**Before posting**:
- Spend 10-15 min engaging with others' content
- Comment thoughtfully on 3-5 posts in your niche
- This primes the algorithm

**After posting**:
- Respond to EVERY comment within first 2 hours
- Ask follow-up questions to commenters
- React to comments (heart, etc.)
- First hour is critical for reach

**Engagement generators**:
- End with specific questions (not "What do you think?")
- Ask for predictions ("I bet this will be 100x by June")
- Challenge readers ("Try this tomorrow, tell me what happens")
- Use polls for high engagement (weekly or bi-weekly)

## Part 5: Automation Opportunity

### Current Manual Process

1. Complete tasks throughout day
2. Log to productivity tracker (CLI or manual CSV)
3. At end of day, export data
4. Manually write LinkedIn post
5. Schedule or post
6. Repeat

### Automated Draft Generation Flow

**Trigger**: Daily at 6:00 PM EST (or on-demand)

**Process**:
1. Read day's productivity data from `.productivity/YYYY-MM-DD.json`
2. Calculate aggregate metrics
3. Select best template based on data (highest ROI task, total count, etc.)
4. Generate draft post using template
5. Send via Telegram with approval buttons
6. Human reviews, edits if needed, approves
7. Post to LinkedIn (via API or manual)

### Technical Implementation

```python
# tools/linkedin_draft_generator.py

from productivity_tracker import ProductivityTracker
from datetime import date
import json

class LinkedInDraftGenerator:
    def __init__(self):
        self.tracker = ProductivityTracker()
        self.templates = self._load_templates()

    def generate_daily_draft(self, target_date=None):
        """Generate LinkedIn draft from day's productivity data."""
        target_date = target_date or date.today()

        # Get day's data
        stats = self.tracker.get_daily_stats(target_date)
        tasks = self.tracker.tasks.get(str(target_date), [])

        if not tasks:
            return None

        # Select template based on data
        template = self._select_template(stats, tasks)

        # Generate draft
        draft = self._fill_template(template, stats, tasks)

        return {
            'date': str(target_date),
            'template_used': template['name'],
            'draft': draft,
            'stats': stats,
            'task_count': len(tasks)
        }

    def _select_template(self, stats, tasks):
        """Choose best template based on data characteristics."""
        # If one task has exceptional ROI, use single task highlight
        max_roi_task = max(tasks, key=lambda t: t.roi_multiplier)
        if max_roi_task.roi_multiplier >= 40:
            return self.templates['single_task_highlight']

        # If many tasks, use daily summary
        if len(tasks) >= 5:
            return self.templates['daily_summary']

        # If it's Friday, use weekly compilation
        if date.today().weekday() == 4:
            return self.templates['weekly_compilation']

        # Default to ROI focus
        return self.templates['roi_focus']
```
