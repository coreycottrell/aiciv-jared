One-Word AI Terminal Setup

Type one word in your terminal. Land directly in your AI's live session. Works with any AI running in tmux on a remote server.

What this does: After setup, you type a single word (like nova or sage) into Terminal on your Mac/PC and it instantly connects you to your AI's live terminal session on your server. Same session as your portal. No passwords, no finding session numbers. Just one word.
Step 1: Give This to Your AI
Copy the prompt below and paste it to your AI. It will set up the server side automatically. Replace the placeholder values with your real info first.

Prompt to give your AI

Copy this entire prompt 
I want to set up a one-word terminal command so I can type a single
word on my Mac and land directly in your live tmux session.

Here's what I need you to do on the server:

1. Create a file called "connect-[YOURAINAME].sh" in your project's
   tools/ directory with this content:

   #!/bin/bash
   PROJECT_DIR="[YOUR_AI_PROJECT_PATH]"
   SESSION=$(cat "$PROJECT_DIR/.current_session" 2>/dev/null)
   if [ -z "$SESSION" ]; then
       echo "No active session found."
       tmux list-sessions 2>/dev/null
       exit 1
   fi
   if tmux has-session -t "$SESSION" 2>/dev/null; then
       exec tmux attach -t "$SESSION"
   else
       echo "Session '$SESSION' not found. Available:"
       tmux list-sessions 2>/dev/null
       exit 1
   fi

2. Make it executable: chmod +x tools/connect-[YOURAINAME].sh

3. Make sure your startup/restart script writes the current tmux
   session name to .current_session every time you start:
   echo "$(tmux display-message -p '#{session_name}')" > .current_session

4. Add a bash function to the server's ~/.bashrc so typing
   "[YOURAINAME]" works when I'm already SSH'd in:

   [YOURAINAME]() {
       local session=$(cat [YOUR_AI_PROJECT_PATH]/.current_session 2>/dev/null)
       if [ -z "$session" ]; then
           echo "No active session found."
           tmux list-sessions 2>/dev/null
           return 1
       fi
       if ! tmux has-session -t "$session" 2>/dev/null; then
           echo "Session '$session' not found."
           tmux list-sessions 2>/dev/null
           return 1
       fi
       if [ -n "$TMUX" ]; then
           tmux switch-client -t "$session"
       else
           tmux attach -t "$session"
       fi
   }

5. Tell me the FULL PATH to the connect script when you're done
   (I need it for the Mac-side setup).
Your AI will create these files and tell you the full path to the connect script. You need that path for Step 2.
Step 2: Run This on Your Mac (One Time)
Save the script below as setup.sh on your Mac. Open Terminal and run: bash setup.sh
It asks you 4 questions, sets up your SSH key (so no password needed), and adds the magic word. Takes about 60 seconds.
setup.sh — save on your Mac, run once 
#!/bin/bash
# ONE-TIME SETUP: Type one word to connect to your AI
# Run once: bash setup.sh

echo ""
echo "=== One-Word AI Terminal Setup ==="
echo ""

read -p "What word do you want to type to connect? (e.g. nova): " CMD
read -p "Server IP address (e.g. 123.45.67.89): " IP
read -p "Server username (e.g. alex): " USER
read -p "Full path to connect script on server: " SCRIPT

echo ""
echo "--- Setting up SSH key (so you never type a password) ---"

if [ ! -f ~/.ssh/id_ed25519 ] && [ ! -f ~/.ssh/id_rsa ]; then
    echo "No SSH key found. Creating one..."
    ssh-keygen -t ed25519 -C "${CMD}-access" -f ~/.ssh/id_ed25519 -N ""
    echo ""
fi

echo "Copying SSH key to server (enter server password when asked)..."
ssh-copy-id "${USER}@${IP}"

echo ""
echo "--- Adding the '${CMD}' command ---"

if [ -f ~/.zshrc ]; then
    RC=~/.zshrc
elif [ -f ~/.bashrc ]; then
    RC=~/.bashrc
else
    RC=~/.zshrc
    touch "$RC"
fi

sed -i.bak "/alias ${CMD}=/d" "$RC" 2>/dev/null

echo "" >> "$RC"
echo "# Type '${CMD}' to connect to your AI" >> "$RC"
echo "alias ${CMD}='ssh -t ${USER}@${IP} bash ${SCRIPT}'" >> "$RC"

echo ""
echo "DONE! Close this terminal, open a new one, and type:"
echo ""
echo "    ${CMD}"
echo ""
echo "To disconnect safely: press Ctrl+B then D"
echo ""
Step 3: Test It
1Close Terminal completely and reopen it
2Type your magic word and press Enter
You should land directly in your AI's live session. If you have a portal, this is the same session.
3To disconnect: press Ctrl+B then D
This detaches you without stopping your AI. It keeps running. You can reconnect anytime with your word.
Troubleshooting
?"command not found"
You need to close and reopen Terminal after setup. Or run source ~/.zshrc
?"No active session found"
Your AI isn't running. Start it first, or ask it to update .current_session in its startup script.
?"sessions should be nested with care"
You're already inside a tmux session. The server-side bash function (Step 1, item 4) handles this — type your AI's name directly instead of using the alias.
?Asks for password every time
SSH key didn't copy correctly. Run ssh-copy-id youruser@your-server-ip again.
The -t flag is critical. If you set this up manually, the SSH alias MUST include -t (forces terminal allocation). Without it, tmux refuses to attach. The setup script handles this automatically.
