Health Check

Is Chy Running?

Follow these steps to check if Chy is alive and working.

1

SSH into the server

Open your Terminal (Mac) or PowerShell (Windows) and run:

ssh -p 2213 aiciv@37.27.237.109
2

Check tmux sessions

Once connected, see what's running:

tmux list-sessions
Healthy output: You should see aiciv-primary listed. That means Chy is running.
No sessions? Chy is not running. Jump to Section 3: Restart Chy.
3

Quick status snapshot

To see a read-only snapshot of what Chy is doing right now without interrupting:

tmux capture-pane -t aiciv-primary -p | tail -40

This shows the last 40 lines of Chy's terminal output.

Check Telegram Bot

Verify the Telegram bridge is running so you can message Chy on your phone:

ps aux | grep telegram_jsonl_monitor | grep -v grep
If nothing shows up, the Telegram bridge is stopped. See Section 4: Services to start it.

Live Monitoring

Attach to Chy's Session (Interactive)

This connects you directly to Chy's live terminal. You can watch and interact.

1

SSH in first

ssh -p 2213 aiciv@37.27.237.109
2

Attach to the tmux session

tmux attach -t aiciv-primary

You are now looking at exactly what Chy sees. You can type commands here.

3

Detach safely (IMPORTANT)

To leave without stopping Chy: Press Ctrl+B then D (the letter D, not Ctrl+D).

This detaches you from the session. Chy continues running in the background.

DO NOT press Ctrl+C or type exit while attached -- that could kill Chy's process.

Read-Only Snapshot (Non-Interactive)

If you just want to peek at what Chy is doing without any risk of accidentally interrupting:

tmux capture-pane -t aiciv-primary -p | tail -60

This captures the visible screen content and prints the last 60 lines. Completely safe -- Chy never knows you looked.

Watch Chy's Log File

For continuous monitoring without attaching to the session:

tail -f /home/aiciv/.claude/memory/summaries/latest.md

Press Ctrl+C to stop watching the log.

Restart Chy

When to Restart

  • Chy is unresponsive or stuck
  • tmux list-sessions shows no aiciv-primary session
  • Claude Code disconnected or OAuth expired
  • After an update you want Chy to pick up

Restart Procedure

Note: Chy runs inside a Docker container. There are no systemd services. Everything runs via tmux.
1

SSH into the server

ssh -p 2213 aiciv@37.27.237.109
2

Kill the existing tmux session (if it exists)

tmux kill-session -t aiciv-primary 2>/dev/null; echo "Session cleared"
3

Start a fresh session

tmux new-session -d -s aiciv-primary "claude --dangerously-skip-permissions"

The -d flag starts it detached so it runs in the background.

4

Verify it started

tmux list-sessions
You should see aiciv-primary listed. Chy is now running.
5

Optionally attach to verify

tmux attach -t aiciv-primary

Watch for a few seconds to confirm Claude Code is loading. Detach with Ctrl+B then D.

One-Liner Restart (Advanced)

Kill and restart in a single command:

tmux kill-session -t aiciv-primary 2>/dev/null; tmux new-session -d -s aiciv-primary "claude --dangerously-skip-permissions" && echo "Chy restarted"

Mac SSH Key Setup

One-Time Setup (Do This Once)

After this setup, you will never need to type a password when connecting to Chy's server.

1

Check if you already have an SSH key

Open Terminal on your Mac and run:

ls ~/.ssh/id_ed25519.pub 2>/dev/null && echo "Key exists!" || echo "No key found"
2

Generate a key (only if "No key found")

If step 1 said "No key found", create one:

ssh-keygen -t ed25519 -C "your-email@example.com"

Press Enter three times to accept defaults (no passphrase for convenience).

3

Copy your key to Chy's server

ssh-copy-id -p 2213 aiciv@37.27.237.109

You will be asked for the server password one final time. After this, all future connections are passwordless.

4

Test the connection

ssh -p 2213 aiciv@37.27.237.109 "echo 'Connected to Chy!'"
If you see "Connected to Chy!" without a password prompt, you are all set.
5

(Optional) Add SSH config for convenience

Add this to ~/.ssh/config on your Mac for easier connections:

Host chy HostName 37.27.237.109 Port 2213 User aiciv

Then you can simply type ssh chy to connect.

One-Word Terminal Setup

The "Chy" Command

After this setup, typing Chy in any Terminal window will SSH into the server and immediately attach to Chy's session.

1

Open your shell profile

On Mac (zsh):

nano ~/.zshrc

On Linux (bash):

nano ~/.bashrc
2

Add this line at the bottom

alias Chy='ssh -t -p 2213 aiciv@37.27.237.109 "bash -lc Chy"'
3

Save and reload

Press Ctrl+X, then Y, then Enter to save nano. Then reload:

source ~/.zshrc
4

Use it

Now, from any Terminal window, just type:

Chy

This runs ssh -t -p 2213 aiciv@37.27.237.109 "bash -lc Chy" behind the scenes, connecting you and attaching to Chy's tmux session automatically.

What the Chy Command Does

When the Chy alias runs on the remote server, it looks for a script or alias called Chy in the server's shell profile. This typically:

  1. Checks if the aiciv-primary tmux session exists
  2. If yes: attaches to it so you can see and interact with Chy
  3. If no: creates a new session running claude --dangerously-skip-permissions
Make sure the Chy alias or script exists on the server at /home/aiciv/.bashrc or /home/aiciv/bin/Chy.

OAuth Reauthorization

When Claude Disconnects

Claude Code uses OAuth to authenticate. Sometimes this expires and needs to be refreshed. Here are the scenarios and how to handle each one.

Scenario A: Claude is running but slow/unresponsive

Symptoms: Chy's tmux session exists but Claude seems stuck or not responding.

Fix: Wait 2-3 minutes. Claude has a processing timeout. If still stuck after 5 minutes, proceed to restart (Section 3).

Scenario B: "Authentication required" or "OAuth expired" error

Symptoms: You see an auth error in the tmux session.

Fix:

# 1. Kill the current session tmux kill-session -t aiciv-primary # 2. Start Claude interactively (not detached) to complete OAuth tmux new-session -s aiciv-primary # 3. Inside tmux, run Claude: claude --dangerously-skip-permissions # 4. Follow the OAuth prompts (open URL in browser, authorize) # 5. Once authenticated, Claude will start normally # 6. Detach with Ctrl+B then D

Scenario C: OAuth prompt shows URL to open

Symptoms: Claude shows a URL and asks you to open it in a browser to authenticate.

Fix:

  1. Copy the URL from the terminal
  2. Open it in your browser on your Mac/phone
  3. Log in with your Anthropic account
  4. Click "Authorize"
  5. Go back to the terminal -- Claude should now continue
Tip: The URL is typically something like https://console.anthropic.com/oauth/.... It expires after a few minutes, so do it promptly.

Scenario D: Nothing works / Persistent auth failure

Symptoms: OAuth keeps failing even after authorizing in browser.

Fix (nuclear option):

# 1. Kill everything tmux kill-session -t aiciv-primary 2>/dev/null # 2. Clear Claude's auth cache rm -rf /home/aiciv/.claude/auth* 2>/dev/null rm -rf /home/aiciv/.config/claude-code/auth* 2>/dev/null # 3. Start fresh tmux new-session -s aiciv-primary # Then run: claude --dangerously-skip-permissions # Complete OAuth flow from scratch
Last resort only. This clears all auth state and requires a full re-authentication. Only do this if Scenarios A-C failed.
