Technical Architecture

Build Blueprint

Hancock Law AI runs on Cloudflare's global edge network with Claude as the intelligence layer. Zero cold starts. Sub-100ms routing. Global availability.

Core Stack

  • Cloudflare Workers -- Edge compute for all API endpoints and agent routing
  • Cloudflare D1 -- SQLite at the edge for legal data, contracts, research, compliance
  • Cloudflare R2 -- Object storage for document uploads (PDFs, DOCX, filings)
  • Claude API (Anthropic) -- Intelligence layer for all agent reasoning
  • Cloudflare Pages -- Static frontend hosting with instant deploys
  • Cloudflare Access -- Zero-trust authentication for enterprise deployments

Data Architecture

D1 Schema for Legal Data

Core Tables

  • legal_contracts -- Contract metadata, parties, risk scores, status tracking
  • legal_clauses -- Extracted clauses with risk level, market percentile, AI recommendations
  • legal_research -- Research queries, answers, citations, jurisdiction tracking
  • legal_compliance -- Regulatory alerts, impact assessments, action tracking
  • legal_templates -- Document templates with customizable variables per jurisdiction
  • legal_matters -- Case/matter management with timelines and assignments
  • legal_billing -- Time entries, narratives, rate analysis, budget tracking
  • legal_memory -- Persistent agent memory per user, per practice area

M365 Integration Points

  • Outlook -- Graph API for email parsing, client comms drafting, deadline extraction
  • Teams -- Bot Framework for conversational agent interface within Teams channels
  • SharePoint -- Document indexing via Graph API, precedent library ingestion
  • Calendar -- Court date tracking, statute of limitations, hearing prep automation
  • OneDrive -- Secure document upload/download for contract review workflows

API Endpoints (per module)

  • POST /api/contracts/upload -- Upload contract for AI review
  • GET /api/contracts/:id/analysis -- Retrieve clause-by-clause analysis
  • POST /api/research/query -- Submit legal research question
  • GET /api/compliance/alerts -- Fetch regulatory change alerts
  • POST /api/documents/generate -- Generate document from template
  • POST /api/billing/narrative -- Generate billing narrative from work log
  • GET /api/matters/:id -- Retrieve matter details and timeline
  • POST /api/conflicts/check -- Run conflict check against firm database
