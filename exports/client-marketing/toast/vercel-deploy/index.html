2.3 Executing the 90-Day Marketing Program

For each client in this phase, the core program is:

Email Marketing (Powered by Toast Data):

  • Month 1: Set up properly segmented email list from Toast guest data. Send welcome sequence to subscribers who have never received an email. Send re-engagement campaign to lapsed guests (60+ days since last visit).
  • Month 2: Launch loyalty-tier email program (different messaging for top 20% spenders vs. first-timers). Execute seasonal campaign for next major holiday or event.
  • Month 3: Measure repeat visit rate change vs. baseline. A/B test subject lines. Document results.

Social Media Content:

  • 3-4 posts per week on Instagram and/or Facebook
  • Content mix: 60% food/experience content, 20% behind-the-scenes/story, 20% promotion/offer
  • Month 1: Establish aesthetic consistency and voice
  • Month 2: Run one paid promotion (even $150-$300 budget) for a specific offer or event
  • Month 3: Evaluate what content drove the most engagement and referral traffic

Google Business Profile:

  • Claim and fully optimize (photos, description, hours, attributes)
  • Set up weekly post cadence
  • Implement review response template and respond to all reviews (positive and negative)
  • Submit for Google local SEO best practices (consistent NAP, keyword-rich description)

Seasonal Event Marketing: Identify 1-2 major restaurant events in the 90-day window (Mother's Day, local food festival, back-to-school, summer kickoff, etc.) and build a full campaign: email sequence, social posts, PR angle if relevant.

Measurement: Track against baseline (establish in Week 1):

  • Email list size and growth
  • Email open rate and click rate
  • Repeat guest rate (from Toast analytics)
  • Google Business Profile views and actions
  • Social followers and engagement rate
  • Revenue on key event days vs. prior year

3.2 Toast API Integration Proof of Concept (Days 100-150)

Goal: Build a working prototype of a Toast-connected marketing automation that demonstrates the product concept, even if it is not yet a polished SaaS product.

Phase 3 Integration Scope (MVP):

The MVP integration should accomplish one thing exceptionally well: automated re-engagement email based on guest lapse data from Toast.

How it works:

  1. Read Toast guest order history via Toast API
  2. Identify guests who visited within the last 6 months but have not returned in 45+ days
  3. Automatically trigger a personalized re-engagement email via the email marketing platform (Brevo, Mailchimp, or similar)
  4. Track whether those guests return within 30 days (close the loop with Toast order data)

This single workflow — which no general marketing agency can execute without the API integration — is a killer demo feature and a genuine revenue driver for clients.

Technical Steps:

  • [ ] Review Toast API documentation at dev.toasttab.com
  • [ ] Obtain API credentials (requires applying for developer access)
  • [ ] Build data extraction script: pull guest visit history, identify lapsed segments
  • [ ] Connect to email platform API to trigger automated sends
  • [ ] Build simple reporting dashboard (even a Google Sheet with formulas is sufficient at MVP stage)
  • [ ] Test with one client's live Toast data (with their written permission)
  • [ ] Document results from first automated campaign

Technical Roles Needed: One developer or technical co-founder to build the integration. This can be outsourced or contracted for Phase 3 MVP. Full-time technical hire is Phase 4.

Estimated Development Cost (MVP): $3,000-$8,000 for a contracted developer to build the initial API connection and email trigger workflow.

4.1 Integration Product Launch (Months 7-10)

Core Feature Set (Version 1.0):

  1. Toast guest data sync (automated, daily updates)
  2. Automated lapse/re-engagement email sequences

  5. Review request automation (trigger post-visit review request to guests with above-threshold spend)

Pricing the Integration Product:

Revenue potential at scale: Even at 1% penetration of Toast's 127,000+ location base = 1,270 locations x $200/month = $254,000/month recurring from the product alone, with near-zero marginal cost.

Our results data shows X% improvement in repeat visit rates across our client base. Sales Playbook

Overview

The sales process for restaurant clients is short when leads are warm (Toast AM referral, restaurant association intro) and medium when cold (direct outreach). Target sales cycle: 7-21 days from first contact to signed agreement.

Step 1: Lead Qualification

Qualifying questions for every prospect:

  1. "How long have you been using Toast?" (Want 12+ months — they have enough data for us to work with)
  2. "How are you currently handling your marketing?" (Listen for: nothing, or doing it myself poorly)
  3. "Are you actively using Toast's email and loyalty features?" (If yes, great — we build on it. If no, even better — there is more opportunity.)
  4. "What does a slow Tuesday look like for you, and what do you do to drive traffic on a night like that?" (Reveals pain and sophistication level)
  5. "If marketing was working for you — really working — what would that look like in your business?" (Reveals their success picture, which we use in the proposal)

Disqualify if:

  • On a non-Toast POS (we cannot deliver our core value)
  • Below $500K revenue (budget constraint)
  • Chain with a marketing department (wrong target)
  • Looking for a one-time website build or logo (not our service)

Step 2: The Toast Marketing Audit (Free)

Offer every qualified prospect a free 30-minute Toast Marketing Audit over video call. During the call:

  1. Ask them to share their screen or grant you view access to their Toast backend
  2. Review the following (takes 10-15 minutes):
    • Guest email list size and engagement status
    • Last marketing email sent (date, subject, open rate if visible)
    • Loyalty program enrollment count (% of guests enrolled)
    • Google Business Profile status (claimed? Review response rate? Recent posts?)
    • Online ordering conversion metrics
  3. Deliver your findings live, conversationally — not a formal presentation
  4. End with: "You have more marketing potential here than you're using. We can help you activate it. Want to see a proposal?"

The audit converts at high rates because:

  • It is free, so the barrier is zero
  • Seeing their own platform through an expert's eyes is compelling
  • The conversation naturally identifies their pain points in their own words, which we mirror back in the proposal

Step 3: The Proposal

Deliver proposals within 24-48 hours of the audit. Format:

  • 2-3 pages maximum
  • Open with their specific situation ("You have 847 guests who ordered online in the last 6 months and have never received an email from you. That is your first campaign.")
  • Recommend a specific tier with rationale
  • Show projected results (conservative, based on industry benchmarks and comparable case studies)
  • Price with all fees itemized
  • Clear next step: "Sign the agreement and we start your onboarding call within 48 hours"

Step 4: Objection Handling

| Objection | Response | |-----------|----------| | "We don't have the budget right now" | "What if we could show you that this program pays for itself? Our typical client sees enough incremental repeat visits in 60 days to cover the monthly fee. Let's start with our Tier 1 and prove the model." | | "I've worked with marketing agencies before and they didn't deliver" | "That is the most common thing I hear. Restaurant operators I work with are managing kitchens, staff, and customers 12 hours a day. Marketing keeps falling to the bottom of the list. We handle it so you don't have to think about it." | | "Can we start smaller?" | "Absolutely. Our Tier 1 at $750/month is designed for exactly that. Toast can give you a kitchen, but it doesn't cook the food." |

Toast API Integration Roadmap

Overview

The Toast API integration is the product that separates this agency from every generalist competitor. It turns raw POS transaction data into automated, personalized marketing. Built in phases, it becomes a standalone SaaS product by Month 12.

Phase 1: Research and Access (Days 100-110)

  • [ ] Review Toast's developer documentation at dev.toasttab.com
  • [ ] Understand available API endpoints: Orders, Guests, Loyalty, Menu, Payments
  • [ ] Apply for Toast developer access (requires business registration)
  • [ ] Understand data privacy requirements (PCI compliance, guest data handling)
  • [ ] Identify which client(s) will serve as the integration pilot (requires written permission)

Key Toast API Endpoints for Marketing Use:

  • GET /orders — Full order history by guest, including spend, items, visit frequency, and recency
  • GET /loyalty/accounts — Loyalty program enrollment and point/tier status by guest
  • GET /guests — Guest profiles with contact info and visit history
  • GET /menus — Menu structure and item performance data (useful for content creation)

Phase 2: MVP Build (Days 110-150)

MVP Scope: One automated workflow, fully functional.

Workflow: Guest Lapse Re-Engagement

[TRIGGER] Guest has not visited in 45 days
     |
[TOAST API] Read last visit date + average spend + typical order type
     |
[SEGMENT] High-value lapsed (avg spend $50+) vs. standard lapsed
     |
[EMAIL PLATFORM] Trigger personalized email
  - High-value: "We miss you. Here's 20% off your next visit."
  - Standard: "Come back this week — here's a treat on us."
     |
[TRACKING] 30-day window: Did they return? Log outcome in reporting.
     |
[REPORT] Monthly: How many lapsed guests were re-engaged? Revenue generated from campaign?

Technical Stack (Recommended):

  • Toast API: Python or Node.js wrapper for API calls
  • Email platform: Brevo (already in use) or Mailchimp via API
  • Database: SQLite or PostgreSQL for storing guest segments and campaign history
  • Reporting: Google Sheets or simple web dashboard

Development timeline: 4-6 weeks for contracted developer

Phase 3: Expansion Features (Months 6-9)

After MVP is validated:

| Feature | Description | Development Estimate | |---------|-------------|----------------------| | Slow-day revenue boost | Identify historically slow days from order history; trigger promotions automatically | 2-3 weeks | | Review request automation | Post-visit email to high-satisfaction guests requesting Google review | 1-2 weeks | | New guest welcome sequence | Trigger 3-email welcome series after first visit detected in Toast | 2-3 weeks | | Loyalty tier campaigns | Automated emails when guest crosses loyalty point threshold | 2-3 weeks | | Menu item recommendation | Based on past orders, recommend new menu items in email content | 3-4 weeks |

Phase 4: Product Launch (Months 10-12)

  • Productize the integration as a standalone SaaS product
  • Build simple onboarding flow (restaurant owner connects their Toast account via OAuth)
  • Create subscription billing infrastructure
  • Submit to Toast Marketplace
  • Price: $149-$249/month standalone; included in Tier 3 managed services

Integration Compliance Notes

  • Toast partner program approval likely required for full API access and Marketplace listing
  • Guest data handling must comply with applicable privacy laws (CAN-SPAM, state privacy laws)
  • Restaurant operators must provide written consent to connect their Toast data to third-party marketing tools
  • PCI compliance considerations for any systems that touch payment data (avoid — use only guest/order data, not payment card data)
