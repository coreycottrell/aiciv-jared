## Finding 4: WordPress "Discourage Search Engines" Setting - PRIME SUSPECT

**What it does**: When enabled at WordPress Settings > Reading, this setting adds:
```html
<meta name='robots' content='noindex,nofollow' />
```
to every page on the site. Some hosting configurations (including some GoDaddy setups) implement this as an HTTP X-Robots-Tag header instead.

**Why it's the prime suspect**:
- Site returns ZERO Google results (site:purebrain.ai = empty)
- This is the #1 most common cause of complete indexing failure on new WordPress sites
- GoDaddy's own help documentation explicitly warns about this setting
- The setting is ON by default on many new WordPress installs
- Even if turned off recently, Google takes days to weeks to re-crawl and re-index

**How to verify (requires WP admin access)**:
1. Log in to WordPress admin (https://purebrain.ai/wp-admin/)
2. Go to Settings > Reading
3. Look for "Search Engine Visibility" checkbox
4. If checked ("Discourage search engines from indexing this site"), this WAS or IS the problem

**Current status**: Unknown from external inspection. The site MAY have had this enabled and it may now be off, OR it may still be on.

## Root Cause Analysis

### Most Likely Root Cause: Site Is Simply Too New / Was Recently Enabled

Google has never crawled purebrain.ai and returned meaningful results. This strongly suggests either:

**Theory A (Most Likely)**: The "Discourage search engines" setting WAS previously enabled and was recently turned off. Google has not yet re-crawled the site after the change. Time for indexing after removing this block: 2-8 weeks passively, or 1-7 days if submitted via Google Search Console.

**Theory B (Also Possible)**: The site has never been submitted to Google Search Console. New sites can take months to be discovered organically without GSC submission.

**Theory C (Less Likely)**: The setting is currently still ON, and the WebFetch tool is not showing the noindex meta tag because it processes HTML differently.
