Use AnimationMixer to blend between them with `action.fadeIn(0.5).play()`.\n\n2. **Scale inconsistency**: Meshy models often export at unexpected scales. Always tune `MODEL_SCALE` constant after first load. Typical adjustment: `scale={0.01}` or `scale={2}`.\n\n3. **UV seams**: Meshy's auto-UV unwrap can produce visible seams on organic shapes. View in Three.js wireframe mode first to check.\n\n4. **Draco + morph targets conflict**: Draco compression CANNOT compress morph target data (blend shapes). If your Meshy model has morph animations, skip Draco or use meshopt instead.\n\n5. **Texture resolution**: Meshy defaults to 1024px textures. For hero elements needing close-up quality, export at 2048px then resize to 1024 after optimization.\n\n6. **HDRI lighting**: Meshy models look best with environment lighting (`<Environment preset=\"city\" />` in Drei). Plain ambient light makes them look flat.\n\n7. **GLB vs GLTF**: Use GLB (binary) for web - everything in one file, better CDN caching, no separate texture requests.\n\n### Sketchfab as an Alternative Pipeline\n\nIf Meshy models need more polish, Sketchfab's Viewer API provides:\n\n```javascript\nconst client = new Sketchfab('your-iframe')\nclient.init('your-model-uid', {\n  success: (api) => {\n    api.start()\n    api.addEventListener('viewerready', () => {\n      // Full programmatic control\n      api.setCameraLookAt([0, 0, 5], [0, 0, 0], 0)\n      api.setMaterial(material, { AlbedoPBR: { color: [1, 0.5, 0] } })\n    })\n  }\n})\n```\n\nSketchfab's viewer API supports annotations, camera movement, material swapping, and animation control -
