## Executive Summary

Three high-performing LinkedIn posts about Claude mastery reveal a content formula that drives significant engagement: **practical tactical lists + clear "before/after" transformation + resource compilation**. Combined engagement: 2,480+ likes, 609+ comments. Key insight: The audience craves *actionable specificity* over conceptual discussion.

## Post Analysis

### Post 1: Charlie Hills - "Basic Chatbot" (801 likes, 391 comments)

**Hook**: "You're using Claude like a basic chatbot"
- Problem/solution framing creates immediate tension
- Implies reader is "doing it wrong" (ego hook)

**Content Structure**:
1. Bold problem statement
2. Six numbered tactical tips
3. Brief explanation per tip (1-2 sentences max)
4. Transformation promise: "mid" to "runs my business"
5. Time claim: "60 seconds" setup
6. Closing question for engagement

**Key Tactics Shared**:
| Feature | Business Value |
|---------|---------------|
| Memory Settings | Retain preferences across conversations |
| Extended Thinking | Step-by-step reasoning for accuracy |
| Claude Opus 4.6 | Most capable model with effort levels |
| Claude Projects | Organized work with file context |
| Skills Installation | Reusable prompts, no repetition |
| Claude Cowork | Multi-step automation (docs, Excel, PDF) |

**Why It Worked**:
- Numbered list (scannable)
- Each tip has immediate application
- Time-based promise (60 seconds)
- Transformation language ("business operating system")

### Post 2: Eric Eden CMO - "Cowork Playbook" (94 likes, 8 comments)

**Hook**: "Claude Cowork just turned 3-hour tasks into 3-minute ones"
- Time transformation is concrete and measurable
- "Actual work" vs "suggestions" differentiator

**Content Structure**:
1. Time transformation hook
2. Ten specific use cases (Desktop Detox, Receipt to Expense Report, etc.)
3. Framework: "3 Rules That Make Cowork Work"
4. Resource offer: 20 prompts in attached PDF
5. Community mention (25K+ subreddit)

**10 Use Cases Documented**:
1. Desktop Detox - file organization
2. Receipt to Expense Report - data extraction
3. Subscription Audit - recurring charge identification
4. Smart File Renamer - naming conventions
5. Project Folder Architect - structure + docs
6. Research Brief Machine - notes to client briefs
7. Deep Researcher - synthesis + gap identification
8. Slide Deck Builder - outlines to presentations
9. Spreadsheet Surgeon - formula fixes
10. Documentation Generator - onboarding materials

**3 Rules Framework**:
1. **Scope it** - single folder, not entire systems
2. **Plan first** - preview actions before execution
3. **Log everything** - summaries and before/after

**Why Lower Engagement vs Charlie Hills**:
- More niche (Cowork-specific)
- Less provocative hook
- PDF attachment may reduce post visibility
- Smaller audience base

## Key Insights for Aether/PureBrain

### 1. Content Format That Works

| Element | Why It Works | Aether Application |
|---------|-------------|-------------------|
| Numbered lists (5-10 items) | Scannable, shareable | Use for AI delegation tips |
| Time transformation claims | Concrete value | "What took 3 hours now takes 3 minutes" |
| Before/After framing | Creates tension | "Using AI like a chatbot vs. running a business" |
| Resource compilations | Share value, build authority | AI tool directories, prompt libraries |
| Engagement questions | Drives comments | "What would you delegate first?" |

### 2. Engagement Drivers

**High Engagement Triggers**:
- Ego hook ("You're doing it wrong")
- Completeness claim ("Everything you need")
- Time savings with specific numbers
- Free value (no paywall on key content)
- Visual hierarchy (bold, bullets, arrows)
- Repost prompts

**Lower Engagement Patterns**:
- PDF attachments (reduces reach)
- Niche feature focus (Cowork alone)
- No question at end

### 3. Tactical Themes Resonating

| Theme | Audience Signal |
|-------|----------------|
| "AI as business operating system" | Beyond chatbot framing |
| Memory and context retention | Continuity matters |
| Multi-step automation | Real work, not suggestions |
| Skills/prompts as reusable assets | Efficiency through systematization |
| Documentation generation | Common pain point |

## Content Ideas for Jared's LinkedIn

Based on these insights, here are content opportunities:

### Idea 1: "You're still using AI assistants. I built an AI team."

**Format**: Problem/solution with numbered list
**Content**:
- Hook: Most people have 1 AI helper. I have 30 specialists.
- List 6-8 specialist agents (security-auditor, pattern-detector, etc.)
- Each with 1-sentence value prop
- Transformation: "Single assistant" to "AI civilization"
- CTA: "What would you delegate if you had a team?"

**Why It Would Work**: Follows Charlie Hills formula, unique angle (AI-CIV concept), concrete numbers (30 agents)

---

### Idea 2: "The 60-Second AI Briefing That Replaced My Morning Chaos"

**Format**: Time transformation + specific workflow
**Content**:
- Hook: Every morning used to be email chaos
- The workflow: Wake-up protocol, memory search, context loading
- Specific time savings with numbers
- Framework: "5 steps to AI-managed mornings"
- CTA: "What's the first thing you'd automate in your morning?"

**Why It Would Work**: Time-based, relatable pain point, numbered steps, transformation language

---

### Idea 3: "I Compiled 50 Industries AI Is Transforming (And How)"

**Format**: Resource compilation
**Content**:
- Reference the master list we already have
- Organized by tier (High Engagement, Decision Makers, Volume)
- Brief insight per category
- Link to longer resource
- CTA: "Which industry should I deep-dive next?"

**Why It Would Work**: Completeness claim, organized categorization, engagement driver

---

### Idea 4: "Stop Prompting. Start Delegating."

**Format**: Contrarian take + framework
**Content**:
- Hook: Prompts are for chatbots. Delegation is for leaders.
- The difference: Prompts = single tasks. Delegation = outcomes.
- 3-rule framework for AI delegation
- Examples from AI-CIV workflow
- CTA: "What's the difference between your best prompt and true delegation?"

**Why It Would Work**: Contrarian hook, framework (3 rules pattern), ties to Director philosophy

---

### Idea 5: "My AI Remembers What Yours Forgot"

**Format**: Problem/solution with technical differentiation
**Content**:
- Hook: Every session, your AI starts fresh. Mine doesn't.
- The problem: Context loss across sessions
- The solution: Memory-first protocol, agent learnings
- Specific numbers: "71% faster with memory"
- CTA: "What would change if your AI truly remembered?"

**Why It Would Work**: Technical differentiation, specific stat, addresses real pain point
