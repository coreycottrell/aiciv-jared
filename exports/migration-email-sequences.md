## Sequence Configuration (Per Sequence)

**Sequence name format**: `migration-drip-[competitor]`

**Examples**:
- `migration-drip-chatgpt`
- `migration-drip-claude`
- `migration-drip-gemini`

**Email timing**:

| Email | Delay from Trigger |
|---|---|
| Email 1 | 0 minutes (immediate) |
| Email 2 | 2 days |
| Email 3 | 4 days |
| Email 4 | 7 days |
| Email 5 | 10 days |

**Exit conditions**:
- Contact purchases PureBrain (tag: `purebrain-customer` applied)
- Contact unsubscribes
- Contact manually removed from sequence

**Re-entry**: Off. A contact should not enter this sequence more than once per competitor. If they take a different competitor's quiz, a separate sequence can trigger (different `from-[competitor]` tag).

## Personalization Variables in Brevo

These variables must be mapped in Brevo's template editor. All emails use `{{FIRSTNAME}}` which maps to Brevo's standard `{{contact.FIRSTNAME}}`.

Optional: If you want to display the specific frustration or use case in any email body, use:
`{{contact.MAIN_FRUSTRATION}}` and `{{contact.PRIMARY_USE_CASES}}`

These are not used in the base email drafts above but can be added as conditional personalization blocks.

## Logo Text Rendering

When building HTML email templates in Brevo, the PureBrain wordmark renders as:

`PUREBR` + `AI` + `N`

Where:
- `PUREBR` = color #2a93c1 (blue)
- `AI` = color #f1420b (orange)
- `N` = color #2a93c1 (blue)

Use `<span>` tags with inline color styles. Do not use an image for the wordmark in email — render it as styled text for deliverability.

## List Assignments

These sequences should apply to contacts already on List 3 (The Neural Feed) OR create a new migration-specific list segment for contacts who have not yet subscribed.

Recommended: Create a Brevo segment called `migration-prospects` filtered by:
- Has tag `migration-intent`
- Does NOT have tag `purebrain-customer`
- Has tag `from-[competitor]` (per sequence)

This segment can be used for reporting on migration sequence performance separately from general Neural Feed subscribers.

## Unsubscribe Handling

All emails must include Brevo's standard unsubscribe footer. Unsubscribing from this sequence should not unsubscribe the contact from all lists — use Brevo's list-level unsubscribe rather than global unsubscribe.

Configure this in Brevo's automation settings under "Email Unsubscribe Behavior."
