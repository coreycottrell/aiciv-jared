# welcome-email-api — CF Worker Technical Specification

**Status:** Ready for implementation
**Date:** 2026-04-23
**Author:** architect-agent
**Replaces:** `send_welcome_email()` in `tools/agentmail_monitor.py`

---

## 1. Problem Statement

The current welcome email flow has four structural defects:

| Defect | Impact |
|--------|--------|
| Template stored at `/tmp/welcome-email-template.html` | Wiped on VPS reboot. Next customer gets no email or fallback garbage. |
| Mixed placeholder formats (`{HUMAN_FIRST}`, `{{HUMAN_FIRST_NAME}}`, `{AI_NAME}`, `{{CIV_NAME}}`) | Mismatch bugs. Already broke for one customer. |
| Gmail SMTP with `GOOGLE_APP_PASSWORD` | Single-threaded, credential-dependent, no delivery receipts, rate-limited. |
| No delivery log | No visibility into what was sent, to whom, when, or whether it succeeded. |

The fix is a dedicated Cloudflare Worker: `welcome-email-api`. One job. Stateless compute. D1-backed templates and logs.

---

## 2. Architecture Overview

```
Trigger Source
(agentmail_monitor.py OR Witness webhook)
            │
            ▼
  POST /send-welcome
  welcome-email-api (CF Worker)
            │
     ┌──────┴──────┐
     │             │
     ▼             ▼
  D1 DB        Brevo API
  (template     (SMTP relay,
   lookup +      not Gmail)
   delivery log)
```

**Key invariants:**
- Worker is fully independent. No imports from social-api, admin-api, meetings-api, or blog-publisher.
- Shares the existing `purebrain-social` D1 database (adds two new tables only).
- No secrets in code. All credentials are CF Worker secrets (`BREVO_API_KEY`, `ADMIN_TOKEN`).
- Deploys to its own CF Worker. A bad deploy here cannot affect any other Worker.

## 5. D1 Schema

```sql
-- schema.sql
-- Run once via: wrangler d1 execute purebrain-social --file=workers/welcome-email-api/schema.sql

CREATE TABLE IF NOT EXISTS email_templates (
  id           TEXT PRIMARY KEY,          -- e.g. "welcome-v1"
  name         TEXT NOT NULL,
  subject_tmpl TEXT NOT NULL,
  html_tmpl    TEXT NOT NULL,
  text_tmpl    TEXT,
  created_at   TEXT DEFAULT (datetime('now')),
  updated_at   TEXT DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS email_delivery_log (
  id              TEXT PRIMARY KEY,         -- UUID generated by Worker
  template_id     TEXT,
  recipient_email TEXT NOT NULL,
  recipient_name  TEXT,
  ai_name         TEXT,
  magic_link      TEXT,
  tier            TEXT,
  status          TEXT DEFAULT 'sent',      -- 'sent' | 'failed' | 'sandbox_redirect'
  sent_at         TEXT DEFAULT (datetime('now')),
  error           TEXT
);
```

## 6. Placeholder Convention (Standardized)

All templates use **exactly four placeholders**, single-brace format only:

| Placeholder | Replaced With | Example |
|-------------|---------------|---------|
| `{HUMAN_FIRST}` | Customer first name | `Jerome` |
| `{AI_NAME}` | Customer's AI name | `Jaimee` |
| `{MAGIC_LINK}` | Portal access URL (already rewritten to .app.purebrain.ai) | `https://jaimee-jerome.app.purebrain.ai/?token=abc123` |
| `{TIER}` | Membership tier | `Awakened` |

No `{{double_brace}}` variants. No `{{HUMAN_FIRST_NAME}}` aliases. The Worker's replacement function applies a single pass of four `replaceAll()` calls.

---

## 7. Default Template Seed

On first deploy, the Worker checks if `welcome-v1` exists in `email_templates`. If not, it seeds the default template (sourced from `exports/portal-files/welcome-email-template-CLEAN.html`) with placeholders substituted to the standard format.

**Subject template:**
```
{AI_NAME} is ready for you, {HUMAN_FIRST} — here's your access link
```

This seed runs at Worker startup via a lightweight init path (not on every request). It is idempotent — safe to re-run.

## 8. API Endpoints

### 8.1 POST /send-welcome

Send a welcome email to a new customer.

**Auth:** None required for this endpoint. The endpoint is not publicly advertised. Caller is either `agentmail_monitor.py` (VPS) or Witness (direct webhook). Rate limiting via Cloudflare WAF is the protection layer if needed.

**Request body:**
```json
{
  "human_email": "customer@example.com",
  "human_first": "Jerome",
  "ai_name": "Jaimee",
  "magic_link": "https://jaimee-jerome.app.purebrain.ai/?token=abc123",
  "tier": "Awakened"
}
```

All fields required. `magic_link` may contain `.ai-civ.com` domain — the Worker rewrites it before sending.

**Sandbox bypass (preserved from agentmail_monitor.py):**
If `human_email` matches `sb-*@*` (sandbox pattern), the Worker redirects delivery to `jared@puretechnology.nyc` and logs `status = 'sandbox_redirect'`. The original sandbox address is logged but never mailed.

**Successful response (200):**
```json
{
  "ok": true,
  "log_id": "01j2xyz...",
  "recipient": "customer@example.com"
}
```

**Failure response (500):**
```json
{
  "ok": false,
  "error": "Brevo API returned 401"
}
```

The delivery log row is written regardless of success or failure. Failed sends log the error in the `error` column.

**Auth:** None.

**Response (200):**
```json
{
  "status": "ok",
  "worker": "welcome-email-api",
  "db": "connected"
}
```

DB connectivity check: runs `SELECT 1 FROM email_templates LIMIT 1`.

### 8.3 GET /templates

**Auth:** `X-Admin-Token` header (same `ADMIN_TOKEN` secret used across portal workers).

**Response (200):**
```json
{
  "templates": [
    {
      "id": "welcome-v1",
      "name": "Default Welcome Email",
      "created_at": "2026-04-23T00:00:00Z",
      "updated_at": "2026-04-23T00:00:00Z"
    }
  ]
}
```

Returns metadata only (no full HTML in list view — prevents giant response payloads).

---

### 8.4 PUT /templates/:id

Update a template. Full replace — caller must supply all fields.

**Auth:** `X-Admin-Token`.

**Request body:**
```json
{
  "name": "Welcome Email v2",
  "subject_tmpl": "{AI_NAME} is ready for you, {HUMAN_FIRST}",
  "html_tmpl": "<!DOCTYPE html>...",
  "text_tmpl": "Hi {HUMAN_FIRST}, your link: {MAGIC_LINK}"
}
```

**Response (200):**
```json
{ "ok": true, "id": "welcome-v1" }
```

The Worker validates that `{HUMAN_FIRST}`, `{AI_NAME}`, and `{MAGIC_LINK}` are present in `html_tmpl` before saving. Missing placeholders return `400` to prevent broken templates from being stored.

## 9. Email Sending: Brevo Transactional API

**Not Gmail SMTP.** Gmail SMTP requires `GOOGLE_APP_PASSWORD`, is single-threaded, rate-limited to ~500/day, and produces no structured delivery receipts. Brevo handles transactional email at any scale with a real API.

**Brevo API call:**
```
POST https://api.brevo.com/v3/smtp/email
Authorization: api-key {BREVO_API_KEY}
Content-Type: application/json
```

**Payload structure:**
```json
{
  "sender": {
    "name": "Aether | PureBrain",
    "email": "purebrain@puremarketing.ai"
  },
  "to": [{ "email": "customer@example.com", "name": "Jerome" }],
  "bcc": [{ "email": "jared@puretechnology.nyc" }],
  "replyTo": { "email": "support@puremarketing.ai" },
  "subject": "Jaimee is ready for you, Jerome — here's your access link",
  "htmlContent": "<!DOCTYPE html>...(rendered template)...",
  "textContent": "Hi Jerome, your link: https://..."
}
```

**BCC to jared@puretechnology.nyc is always present.** This replaces the Telegram notification about email sends — Jared gets a copy of every welcome email.

## 10. Domain Rewrite

If `magic_link` contains `.ai-civ.com`, the Worker rewrites it to `.app.purebrain.ai` before sending:

```
https://jaimee-jerome.ai-civ.com/?token=abc123
→ https://jaimee-jerome.app.purebrain.ai/?token=abc123
```

Single string replacement inside the Worker. This is the same rewrite that `handle_magic_link_email()` in `agentmail_monitor.py` already performs — it moves to the Worker so the rewrite is guaranteed regardless of what the caller sends.

## 11. Worker Code Structure

```javascript
// workers/welcome-email-api/src/worker.js

// --- Constants ---
const DOMAIN_REWRITE_FROM = '.ai-civ.com';
const DOMAIN_REWRITE_TO   = '.app.purebrain.ai';
const SANDBOX_PATTERN      = /^sb-[^@]+@/i;
const SANDBOX_REDIRECT     = 'jared@puretechnology.nyc';
const BREVO_SEND_URL       = 'https://api.brevo.com/v3/smtp/email';
const DEFAULT_TEMPLATE_ID  = 'welcome-v1';

// --- Router ---
export default {
  async fetch(request, env) {
    await ensureSchema(env);   // idempotent seed on cold start

    const url    = new URL(request.url);
    const method = request.method;
    const path   = url.pathname;

    if (method === 'OPTIONS') return cors();

    if (method === 'POST' && path === '/send-welcome')    return handleSend(request, env);
    if (method === 'GET'  && path === '/health')          return handleHealth(env);
    if (method === 'GET'  && path === '/templates')       return requireAuth(request, env, () => handleListTemplates(env));
    if (method === 'PUT'  && path.startsWith('/templates/')) return requireAuth(request, env, () => handlePutTemplate(request, env, path));
    if (method === 'GET'  && path === '/delivery-log')   return requireAuth(request, env, () => handleDeliveryLog(request, env));

    return err(404, 'Not found');
  }
};
```

**Function responsibilities:**

| Function | Does |
|----------|------|
| `ensureSchema(env)` | CREATE TABLE IF NOT EXISTS + seed default template. Runs on every cold start, no-ops if tables exist. |
| `handleSend(request, env)` | Validate payload, sandbox check, domain rewrite, load template from D1, replace placeholders, call Brevo, write delivery log row. |
| `handleHealth(env)` | SELECT 1 from D1, return ok/error. |
| `handleListTemplates(env)` | SELECT id, name, created_at, updated_at FROM email_templates. |
| `handlePutTemplate(request, env, path)` | Validate placeholders present, UPDATE or INSERT template row. |
| `handleDeliveryLog(request, env)` | Paginated SELECT from email_delivery_log with optional status filter. |
| `requireAuth(request, env, handler)` | Check X-Admin-Token against env.ADMIN_TOKEN. Returns 401 or calls handler. |
| `rewriteDomain(link)` | String replace .ai-civ.com → .app.purebrain.ai. |
| `isSandbox(email)` | Regex check for sb-* pattern. |
| `renderTemplate(html, vars)` | Four replaceAll() calls. Returns rendered string. |
| `sendBrevo(env, payload)` | Fetch to Brevo API. Returns { ok, status, body }. |
| `logDelivery(env, row)` | INSERT INTO email_delivery_log. Fire-and-forget (does not block response). |

## 12. Integration: Phase 1 (agentmail_monitor.py changes)

Minimal change to the existing Python script. Replace the `send_welcome_email()` call with an HTTP POST to the Worker:

```python
WELCOME_EMAIL_WORKER = "https://welcome-email-api.workers.dev/send-welcome"
# (or custom domain: https://welcome.purebrain.ai/send-welcome once DNS set)

def send_welcome_email_via_worker(human_email, human_first, ai_name, magic_link, tier="Awakened"):
    payload = {
        "human_email": human_email,
        "human_first": human_first,
        "ai_name": ai_name,
        "magic_link": magic_link,
        "tier": tier,
    }
    try:
        r = requests.post(WELCOME_EMAIL_WORKER, json=payload, timeout=15)
        data = r.json()
        if data.get("ok"):
            log.info(f"Welcome email dispatched via Worker: log_id={data.get('log_id')}")
        else:
            log.error(f"Worker refused welcome email: {data.get('error')}")
    except Exception as e:
        log.error(f"Failed to reach welcome-email-api Worker: {e}")
```

The old `send_welcome_email()` function and the `/tmp` template dependency are removed. The dual-email case (PayPal + chatbox) calls this function twice — one call per address, same as before.

## 17. Security Notes

- `/send-welcome` has no auth. It is an internal endpoint not exposed via portal or public docs. Cloudflare WAF can restrict by IP (VPS IP + Witness IP) if needed.
- `ADMIN_TOKEN` for management endpoints follows the same pattern as `meetings-api` and `admin-api`. Same value can be reused.
- `BREVO_API_KEY` is a CF Worker secret — never committed to git, never in `wrangler.toml`.
- Template HTML is stored in D1, not in the Worker code. XSS is not a risk since the HTML is only sent outbound as email, never rendered by a browser under our control.

1. **Brevo account confirmed?** Spec assumes Brevo transactional API is available. If not yet set up, coder-agent needs the API key from Jared before deployment.
2. **Custom domain for Worker?** `welcome-email-api.workers.dev` works immediately. `welcome.purebrain.ai` requires a CNAME in CF DNS — low effort, cleaner URLs.

4. **Brevo sender domain verified?** `purebrain@puremarketing.ai` must be verified in Brevo's domain settings or emails land in spam.
