## EXECUTIVE SUMMARY

PureBrain.ai stands at an inflection point. The product is built. Full onboarding automation is imminent. Three enterprise prospects are in conversation pre-launch. A 5-year financial model projects $100M ARR by Year 5 (base case) with $74.3M ARR at 17,200 customers and 68 employees.

The question is no longer "can we build this?" It is "how fast can we scale?"

This plan defines the three initiatives that will determine PureBrain's trajectory over the next 12 weeks and beyond:

| Priority | Objective | Success Metric (90 Days) |
|----------|-----------|--------------------------|
| **1. Investor Strategy** | Create an AI-powered investor experience and close a funding round | 20+ personalized investor pages live, 10+ meetings booked |

These three priorities are mutually reinforcing. The investor page (Priority 1) uses the same interactive technology that differentiates the product (Priority 3). Voice AI (Priority 3) creates the product differentiation that drives organic demand (Priority 2).

## PRIORITY 3: VOICE AI + CONVERSATIONAL AI IN PORTAL

### 3.1 The Vision

Add voice interaction and AI avatars to the PureBrain portal so customers can talk to their PureBrain and their PureBrain talks back. This transforms the portal from a text chat interface into a conversational AI experience. It is the most visible differentiator in the market: most AI tools are text-only. PureBrain becomes the first AI partner you can see and hear.

### 3.2 Feature Phases

**Phase 1: Voice Input/Output (Weeks 1-4)**
*Ship fast. Zero-to-low API cost. Immediate product differentiation.*

| Feature | Implementation | Cost | Difficulty |
|---------|---------------|------|------------|
| Microphone button in portal chat | Web Speech API (browser-native STT) | Free | Low |
| Speech-to-text for user input | Web Speech API or OpenAI Whisper API | Free or ~$0.006/min | Low-Medium |
| Text-to-speech for PureBrain responses | Browser TTS (free) or ElevenLabs/OpenAI TTS | Free or ~$0.015-$0.30/1K chars | Low-Medium |
| Read-aloud button on responses | Browser SpeechSynthesis API | Free | Low |
| Push-to-talk mode | Client-side implementation | Free | Low |
| Hands-free / continuous listening mode | Voice Activity Detection (VAD) | Free | Medium |

**Quick Wins (Ship This Week):**
1. Microphone button in portal chat (Web Speech API, zero cost)
2. Read-aloud button on PureBrain responses (browser TTS, zero cost)

These two features alone make the portal feel significantly more modern and accessible. They require no API costs and can ship in days.

### 3.3 Technical Architecture

```
User's Browser
    |
    |-- Microphone Input
    |       |
    |       v
    |   Voice Activity Detection (VAD) -- client-side, detects speech start/stop
    |       |
    |       v
    |   Speech-to-Text (STT)
    |       |-- Option A: Web Speech API (free, browser-native, Chrome/Edge)
    |       |-- Option B: OpenAI Whisper API (better accuracy, $0.006/min)
    |       |
    |       v
    |   Text sent to PureBrain API (same as chat, just voice-originated)
    |       |
    |       v
    |   PureBrain Response (text)
    |       |
    |       v
    |   Text-to-Speech (TTS)
    |       |-- Option A: Browser SpeechSynthesis API (free, lower quality)
    |       |-- Option B: OpenAI TTS API ($0.015/1K chars, good quality)
    |       |-- Option C: ElevenLabs ($0.30/1K chars, best quality)
    |       |
    |       v
    |   Audio Output + Avatar Animation (lip sync from TTS phonemes)
    |       |
    |       v
    |   Speaker / Headphones
    |
    |-- Avatar Rendering (Three.js)
            |-- Idle animation (breathing, blinking)
            |-- Lip sync (mapped from TTS output)
            |-- Expression changes (context-driven)
            |-- Gesture library (nodding, thinking, presenting)
```

**Streaming Architecture (Critical for Low Latency):**

Do not wait for the full PureBrain response before starting TTS. Stream the response:

1. PureBrain API returns response via Server-Sent Events (SSE)
2. As each sentence completes, send to TTS API
3. TTS audio streams back and plays immediately
4. Avatar lip-syncs in real-time to streaming audio

**Result**: First audio plays within 1-2 seconds of user finishing speech, vs. 5-10 seconds if waiting for full response.

### 3.4 Cost Modeling for Voice Features

| Usage Scenario | STT Cost | TTS Cost | Total/Conversation | Monthly (100 active voice users) |
|---------------|----------|----------|-------------------|----------------------------------|
| **Free tier** (Browser APIs) | $0 | $0 | $0 | $0 |
| **Standard** (Whisper + OpenAI TTS) | $0.006/min | $0.015/1K chars | ~$0.05/conversation | ~$150/mo |
| **Premium** (Whisper + ElevenLabs) | $0.006/min | $0.30/1K chars | ~$0.35/conversation | ~$1,050/mo |

**Recommendation**: Default to browser-native APIs (free). Offer premium voice (ElevenLabs quality) as a Partnered+ tier feature. This creates a natural upsell path.
