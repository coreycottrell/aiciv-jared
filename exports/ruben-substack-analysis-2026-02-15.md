## 1. The Unique Concept/Approach

### Core Thesis
Personal "voice" is not mystical. It's a reproducible pattern that can be documented, transferred, and scaled via AI.

### The Framework: 100-Question Self-Interview

Hassid created a "Taste Interviewer" prompt that extracts someone's communication DNA across six categories:

| Category | Questions | Purpose |
|----------|-----------|---------|
| **Beliefs & Contrarian Takes** | 15 | Unconventional viewpoints, "hot takes you'd defend to the death" |
| **Writing Mechanics** | 20 | Sentence structure, vocabulary preferences, opening/closing techniques |
| **Aesthetic Crimes** | 15 | Phrases that make you cringe, content types you despise |
| **Voice & Personality** | 15 | Humor deployment, tone shifts, controversy handling |
| **Structural Preferences** | 15 | How you organize ideas, use lists/headers, transition between concepts |
| **Hard Nos & Red Flags** | 20 | Topics you'd never write about, untouchable boundaries |

### The Process

1. Use Claude's "Cowork" feature with Opus model
2. Run the 100-question interview prompt
3. Generate comprehensive markdown profile
4. Upload `.md` file to any AI with each prompt
5. AI now writes in your exact voice

### Philosophical Reframe

The potentially disturbing realization ("I'm just a text file") gets recontextualized as liberation:

> "Master yourself so AI can master you" (rather than fearing replacement)

Self-knowledge becomes competitive advantage, not existential threat.

## 3. Technical Implementation Details

### Required Tools
- Claude with Cowork feature (Opus 4.5 recommended)
- Markdown editor
- Any AI platform that accepts file uploads

### The Interview Protocol

The 100-question interview follows specific rules:

1. **One question at a time** - No rapid-fire lists
2. **Push back on vague answers** - "What specifically do you mean?"
3. **Request concrete examples** - "Show me a sentence you'd write"
4. **Identify contradictions** - "Earlier you said X, but now Y"
5. **Explore emerging threads** - Don't accept "I don't know"

### Output Format

The resulting markdown file contains:

```markdown
# [Name]'s Voice Profile

## Beliefs & Positions
- [Specific contrarian take]
- [Another strong opinion]

## Writing Mechanics
- Sentence length preference: [short/medium/long]
- Opening style: [how you start pieces]
- Vocabulary: [words you use, words you avoid]
- Punctuation: [em dashes, parentheses, etc.]

## Aesthetic Crimes
- Never say: [phrases that make you cringe]
- Avoid: [patterns you find lazy]

## Voice Calibration
- Humor: [when/how you deploy it]
- Seriousness: [when you get serious]
- Controversy: [how you handle hot topics]

## Structural Defaults
- How you organize: [lists, narratives, etc.]
- Transitions: [how you move between ideas]

## Hard Boundaries
- Never write about: [topics off-limits]
- Red flags: [things that signal inauthenticity]
```

### Usage

Attach the .md file to any prompt:
```
[Your voice file attached]

Write a LinkedIn post about [topic] in my voice.
```

The AI adapts to your documented patterns instead of generic defaults.
