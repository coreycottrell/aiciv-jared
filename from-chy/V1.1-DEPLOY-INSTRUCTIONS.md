## CHANGES (surgical)
1. **`handleCreatePartner`** (around line 1050) — now accepts `mode: 'poll' | 'webhook'` (default poll). For poll, mints a `pp_<48char>` bearer stored hashed in `ai_partners.poll_token_hash`, returned ONCE in `poll_credentials.token` at registration.
2. **NEW: `resolvePollPartner`** helper — validates poll bearer + marks `last_polled_at`.
3. **NEW: `handlePollWork`** — GET /api/ai_partners/{id}/jobs → atomic claim next pending from `partner_jobs` table, returns job or 204.
4. **NEW: `handlePollResults`** — POST /api/ai_partners/{id}/results → marks completed, side-effect: generate_week results get auto-inserted into `content_items` as drafts (same behavior as current webhook path).
5. **Route dispatcher** updated — matches `/api/ai_partners/{id}/jobs` (GET) + `/api/ai_partners/{id}/results` (POST).
