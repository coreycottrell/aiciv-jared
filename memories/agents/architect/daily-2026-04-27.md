3. Three-phase scaling plan for D1 limits: Phase 1 single DB (0-10M), Phase 2 shard by clause_type (10-50M), Phase 3 R2 for text + vector DB (50M+)

5. User memory table (user_memory) as key differentiator — no other legal tech platform has persistent memory of contract style preferences
6. Multi-agent review pattern (legal + security + compliance in parallel) as second major differentiator
7. Word add-in via Office JS, served from CF Pages — no separate hosting needed
8. Ingestion target: 100K clauses night 1, 1M week 1, 10M month 1 via Cloudflare CRON triggers on ingestion-worker
