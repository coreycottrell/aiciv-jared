### Key Findings (Task 1)

#### Architecture
Two-layer design:
1. social-api worker = management + auth proxy (user session cookie auth)
2. trio-comms worker = message store (bearer token auth, multi-tenant via trio_id)

#### Bugs Found
- handleUpload calls auth() (undefined) instead of authSender() — upload endpoint broken
- TRIO_COMMS_TOKEN means all portal human messages land as a fixed identity, not "jared"
- Viewer role exists in schema but is never enforced
- Portal proxy hardcodes limit=200 with no since forwarding

#### Multi-tenancy
trio_id column added April 2026. All legacy messages default to 'trio-0'. New customers get unique trio IDs. No setup needed — just start using a new trio_id value.

#### AI delivery
Primary injector polls every 20s, injects via tmux send-keys -l + 5x Enter (0.3s gaps).
Auto-responder AFK fallback triggers after 5 min of Primary silence.
