# Google Search Console Setup Guide for purebrain.ai

**Goal**: Get purebrain.ai indexed on Google (currently showing ZERO results)
**Time required**: About 30-45 minutes
**Device**: Works on phone or desktop

## Part 1: Access Google Search Console

**URL**: https://search.google.com/search-console

1. Open that URL in your browser
2. Sign in with your Google account - use the same Google account connected to your business (ideally the one connected to Google Analytics if you have it)
3. If you have never been here before, you will see a welcome screen asking you to add a property

---

## Part 2: Add purebrain.ai as a Property

When you first land on Search Console (or click "+ Add property" in the top-left dropdown):

1. You will see two options:
   - **Domain** (recommended)
   - **URL prefix**

2. Choose **Domain**

3. In the text field, type: `purebrain.ai` (no https, no www - just the bare domain)

4. Click **Continue**

5. Google will show you a TXT record to add to your DNS. It looks like:
   ```
   google-site-verification=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
   **Copy this entire string** - you will need it in the next step.

---

## Part 3: Verify Ownership via Cloudflare DNS

Since purebrain.ai uses Cloudflare, this is the fastest verification method.

1. Open a new tab and go to: https://dash.cloudflare.com
2. Log in to your Cloudflare account
3. Click on **purebrain.ai** in your domain list
4. Click **DNS** in the left sidebar (or top navigation)
5. Click the **+ Add record** button
6. Fill in these fields:
   - **Type**: TXT
   - **Name**: @ (just the @ symbol, which means the root domain)
   - **Content**: Paste the full verification code Google gave you (the `google-site-verification=xxxxx` string)
   - **TTL**: Auto
7. Click **Save**
8. Go back to the Google Search Console tab
9. Click **Verify**

DNS changes usually take between 15 minutes and a few hours to propagate. If verification fails, wait 30 minutes and try the Verify button again.

Once verified, you will see: "Ownership verified" and be taken into your Search Console dashboard.

## Part 4: Submit Your Sitemaps

Your site has 4 sitemaps generated by Yoast SEO. Submit the index first, then the individual ones as backups.

**In Search Console:**

1. In the left sidebar, click **Indexing** (it may say just "Index" on mobile)
2. Click **Sitemaps**
3. You will see a box that says "Add a new sitemap"

**Submit these one at a time** - paste each URL, click Submit, wait for the green checkmark, then do the next one:

**Sitemap 1 (do this one first):**
```
sitemap_index.xml
```
(Just type the filename - Search Console already knows your domain)

**Sitemap 2:**
```
post-sitemap.xml
```

**Sitemap 3:**
```
page-sitemap.xml
```

**Sitemap 4:**
```
category-sitemap.xml
```

After submitting, each sitemap should show a green "Success" status. If you see an error, wait a few minutes and refresh the page.

## Part 5: Request Indexing for Priority Pages

This step tells Google "crawl these pages NOW" instead of waiting for Google to discover them naturally.

**In Search Console:**

1. At the very top of the page, there is a search bar that says "Inspect any URL in..."
2. For each URL below, paste it in that bar and press Enter

**Do these URLs in order:**

### Homepage
```
https://purebrain.ai/
```
- After pasting and pressing Enter, wait for the analysis to load
- Click the button that says **"Request Indexing"**
- Wait for the confirmation popup, click OK
- Repeat for each URL below

### Blog listing page
```
https://purebrain.ai/blog/
```
Request indexing.

### Assessment page
```
https://purebrain.ai/ai-adoption-assessment/
```
Request indexing.

### Blog posts - find these from your blog page
Go to https://purebrain.ai/blog/ and note each blog post URL. Then come back to Search Console and request indexing for each one. They will look something like:
```
https://purebrain.ai/blog/[post-slug]/
```
Request indexing for each blog post you find.

**Note**: Google has a daily limit on manual indexing requests (approximately 10-20 per day for new sites). Prioritize the homepage, blog listing, and assessment page first.

## Part 6: Noindex Your Test Pages

These pages should NOT appear in Google search results. You need to mark them as noindex in WordPress so Google knows to skip them.

**Pages to noindex:**
- purebrain-2-0 (or similar old version pages)
- purebrain-3
- purebrain-4
- living-avatar
- pay-test
- pay-test-sandbox

**How to do it for each page (using Yoast SEO):**

1. Log in to WordPress admin at https://purebrain.ai/wp-admin
2. Go to **Pages** in the left sidebar
3. Find the first test page (e.g., pay-test) and click **Edit**
4. Scroll down below the page content until you see the **Yoast SEO** box
5. Click the **Advanced** tab inside the Yoast SEO box
6. Find the field: "Allow search engines to show this post in search results?"
7. Change the dropdown from "Yes" to **No**
8. Click **Update** (the blue publish/update button)
9. Repeat for each test page

After setting noindex, these pages will be excluded from Google's search results within a few days.

**Also check this setting** (this sometimes causes ALL pages to be blocked):
1. In WordPress admin, go to **Settings** > **Reading**
2. Make sure the checkbox "Discourage search engines from indexing this site" is **NOT checked**
3. If it is checked, uncheck it and click Save Changes
